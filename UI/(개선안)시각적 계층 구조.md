
## 📌 목표

진단 메시지의 중요도를 시각적으로 구분하여 사용자가 우선순위를 직관적으로 파악

## 🎯 접근 방법

## A. 데이터 모델 확장

- `DiagnosticItem`에 **심각도(Severity) 속성** 추가
    
- `enum`으로 `Info`, `Warning`, `Error`, `Critical` 정의
    
- 파싱 단계에서 체커 이름 기반으로 심각도 자동 분류
    

## B. 시각적 표현 전략

1. **색상 코딩**
    
    - 빨강: 컴파일 오류 (즉시 수정 필요)
        
    - 주황/노랑: 정적 분석 경고
        
    - 회색: 참고 정보
        
2. **폰트 스타일**
    
    - 굵기: 심각도에 비례
        
    - 기울임: 부가 정보 표시
        
3. **아이콘/기호**
    
    - 이모지 또는 Material Design 아이콘 활용
        
    - 심각도별로 다른 아이콘 매핑
        

## C. XAML 구현 방향

- **Option 1**: DataTrigger - 간단하지만 XAML이 길어짐
    
- **Option 2**: ValueConverter - 재사용 가능하고 유연함
    
- **Option 3**: TemplateSelector - 복잡하지만 완전히 다른 레이아웃 가능
    

## D. 추가 개선 아이디어

- 심각도별 **그룹화** (ListView를 GroupStyle로 구성)
    
- **배지(Badge)** 로 각 심각도별 개수 표시
    
- **정렬 옵션**: 심각도순, 파일명순, 라인 번호순
