

## 📌 목표

사용자가 보고 싶은 진단 메시지 유형만 선택적으로 표시

## 🎯 접근 방법

## A. 필터 설정 데이터 모델

- `AnalysisOptions` 또는 `FilterSettings` 클래스 생성
    
- 각 필터 항목을 `bool` 속성으로 정의
    
    - `ShowCompilerErrors`
        
    - `ShowCompilerWarnings`
        
    - `ShowDefenceStyleChecks`
        
    - `ShowRelatedInfo`
        

## B. 필터링 구현 위치

**Option 1: 파싱 단계 필터링 (비추천)**

- `ParseDiagnosticLine`에서 조건 검사 후 `null` 반환
    
- 단점: 필터 변경 시 재분석 필요
    

**Option 2: ViewModel 필터링 (추천)**

- 모든 진단 항목을 저장
    
- `ObservableCollection`과 별도의 `FilteredDiagnostics` 컬렉션 사용
    
- 필터 변경 시 실시간 UI 업데이트
    

**Option 3: CollectionViewSource 필터링 (가장 추천)**

- WPF의 `CollectionViewSource.Filter` 활용
    
- MVVM 패턴에 이상적
    
- 성능 최적화 및 정렬/그룹화와 통합 가능
    

## C. UI 구성 방향

1. **체크박스 그룹**
    
    - 상단 툴바 또는 사이드 패널 배치
        
    - 각 심각도/체커 타입별 ON/OFF 토글
        
2. **드롭다운 메뉴**
    
    - "필터" 버튼 → 하위 메뉴로 옵션 표시
        
    - 공간 절약
        
3. **QuickFilter 버튼**
    
    - "오류만 보기", "모두 보기", "defence-style만" 등 프리셋
        
    - 자주 사용하는 필터 조합을 원클릭으로
        

## D. 필터 상태 저장

- JSON/XML 설정 파일로 저장
    
- 앱 재시작 시 마지막 필터 설정 복원
    
- 사용자별 프로필 지원 가능
    

## E. 추가 고급 필터링

- **텍스트 검색**: 메시지 내용 필터링
    
- **파일 경로 필터**: 특정 디렉토리만 표시
    
- **체커 이름 다중 선택**: 원하는 체커만 조합
    
- **정규식 필터**: 고급 사용자용
