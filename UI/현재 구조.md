
```scss
SentinelStatic_Application/
│
├── Views/ (UI 계층 - 사용자에게 보여지는 화면)
│   │
│   ├─ MainWindow.xaml
│   │  └─ 역할: 애플리케이션의 메인 창 레이아웃 정의 (프로젝트 설정, 체커 목록, 분석 결과 그리드 등)
│   │
│   ├─ MainWindow.xaml.cs
│   │  └─ 역할: MainWindow의 코드 비하인드. MainViewModel을 생성하여 DataContext에 연결하는 등                View와 ViewModel을 잇는 최소한의 로직 수행
│   │
│   ├─ CheckerOptionsWindow.xaml
│   │  └─ 역할: 개별 체커의 세부 옵션을 설정하는 팝업 창의 UI 레이아웃 정의
│   │
│   └─ CheckerOptionsWindow.xaml.cs
│      └─ 역할: CheckerOptionsWindow의 코드 비하인드. 창을 보여주는 로직 수행
│
├── ViewModels/ (뷰모델 계층 - View를 위한 데이터와 로직 처리)
│   │
│   └─ MainViewModel.cs
│      └─ 역할: 애플리케이션의 핵심 두뇌. UI에 표시될 데이터(파일 목록, 체커 목록)와 상태(분석 중 여부)를 관리하고, 사용자 명령어(분석 실행, 파일 로드)를 처리하며, 각종 서비스를 호출함
│
├── Models/ (모델 계층 - 애플리케이션에서 사용하는 데이터의 구조 정의)
│   │
│   ├─ AppSettings.cs
│   │  └─ 역할: 애플리케이션의 설정을 파일로 저장하거나 불러올 때 사용하는 데이터 클래스
│   │
│   ├─ CheckerItem.cs
│   │  └─ 역할: Clang-Tidy 체커 하나의 상태(이름, 선택 여부)와 옵션 목록을 담는 데이터 클래스
│   │

│   │  └─ 역할: CheckerItem에 속한 개별 옵션의 상태(이름, 선택 여부)를 담는 데이터 클래스
│   │
│   ├─ SourceFileItem.cs
│   │  └─ 역할: 분석 대상이 되는 소스 파일 하나의 정보(경로, 선택 여부)를 담는 데이터 클래스
│   │
│   └─ DiagnosticItem.cs
│      └─ 역할: Clang-Tidy 분석 결과로 나온 진단 메시지 하나의 정보(파일, 라인, 메시지 등)를 담는 데이터 클래스

    │  └─ 역할: clang-tidy.exe 프로세스를 실행하고, 설정 파일을 동적으로 생성하며, 출력 결과를 파싱하여 ViewModel에 전달
    │
    ├─ ICheckerProviderService.cs / CheckerProviderService.cs
    │  └─ 역할: 애플리케이션이 지원하는 기본 Clang-Tidy 체커 목록과 각 체커의 기본 옵션을 생성하여 제공
    │
    ├─ IFileService.cs / FileService.cs
    │  └─ 역할: 지정된 프로젝트 경로에서 분석할 소스 파일(.c, .cpp)을 검색하는 기능 제공
    │
    └─ ISettingsService.cs / JsonSettingsService.cs
       └─ 역할: AppSettings 객체를 JSON 파일 형태로 저장하고 불러오는 기능 제공
```

1. **시작**: `MainWindow`가 생성되고, `MainWindow.xaml.cs`에서 `MainViewModel`을 초기화합니다.MainWindow.xaml.cs
    
2. **초기화**: `MainViewModel`은 `JsonSettingsService`를 통해 이전 설정을 불러오고, `CheckerProviderService`를 통해 기본 체커 목록을 `Checkers` 컬렉션에 채웁니다.MainViewModel.cs
    
3. **사용자 설정**: 사용자가 `ProjectPath`를 입력하고 'Reload File List' 버튼을 클릭합니다.
    
4. **파일 로드**: `MainViewModel`의 `LoadSourceFilesCommand`가 실행되어 `FileService`를 호출, 소스 파일 목록을 `SourceFiles`에 채웁니다.MainViewModel.cs
    
5. **분석 실행**: 사용자가 분석할 파일과 체커를 선택한 후 'Run Analysis' 버튼을 클릭합니다.
    
6. **`clang-tidy` 실행**: `MainViewModel`의 `RunAnalysisCommand`가 `ClangTidyRunnerService`의 `RunAnalysisAsync`를 호출합니다.MainViewModel.cs
    
7. **설정 파일 생성 및 실행**: `ClangTidyRunnerService`는 선택된 체커 정보로 `.clang-tidy` 파일을 생성하고 `clang-tidy.exe`를 실행합니다.ClangTidyRunnerService.cs
    
8. **결과 수신 및 표시**: `clang-tidy.exe`가 출력하는 진단 메시지를 실시간으로 받아 `MainViewModel`의 `Diagnostics` 컬렉션에 추가하고, 이는 데이터 바인딩을 통해 `MainWindow`의 `DataGrid`에 자동으로 표시됩니다.MainViewModel.cs
    
9. **종료 및 저장**: 사용자가 창을 닫으면 `Window_Closing` 이벤트가 발생하여 `JsonSettingsService`를 통해 현재 체커 선택 상태 등의 설정을 파일에 저장합니다.
