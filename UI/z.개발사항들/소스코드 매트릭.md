
### 국방 무기체계 필요 메트릭 종류 

- Cyclomatic Complexity (순환복잡도)

- Number of Call Levels (함수 내 최대 중첩(Nesting) 깊이)

- Number of Function Parameters (파라미터 개수)

- Number of Calling Functions (FanIN: **나를 호출하는** 함수종류의 개수)

- Number of Called Functions (FanOUT: **내가 호출하는** 함수종의 개수)

- Number of Executable Code Lines (실행문이 포함된 라인 수)

## function-code-metric

- **순환 복잡도(Cyclomatic Complexity)** $E - N + 2$

- **최대 중첩 깊이 (`NestingThreshold`)** 함수 내부에서 `if`문이나 반복문 등의 중괄호 `{}` 블록이 얼마나 깊게 파고드는지(들여쓰기 단계) 검사합니다.

- **매개변수 개수 (`ParameterThreshold`)** 함수가 전달받는 인자(Parameter)의 총 개수를 확인합니다.

- **공백/주석 포함 전체 라인 수 (`LineThreshold`)** 함수의 시작 `{` 부터 끝 `}` 까지, 빈 줄과 주석을 모두 포함한 물리적인 총 줄 수를 측정합니다.

- **실행 가능한 순수 코드 라인 수 (`NclocThreshold`) `Lexer`를 활용해 주석과 빈 줄을 완벽히 제외하고, 실제 유효한 코드가 존재하는 물리적 줄 수(NCLOC)만 정확하게 카운트합니다.

- **구문 개수 (`StatementThreshold`)** 세미콜론(`;`)으로 끝나는 문장이나 `if`, `for` 등의 제어문 등 논리적인 실행 단위(명령어)의 개수를 셉니다.

- **지역 변수 개수 (`VariableThreshold`)** 함수 내부에서 선언된 지역 변수가 너무 많지 않은지 그 개수를 셉니다.


```cpp
void Callee() { 
    // 호출을 당하는 쪽 (나)
}

void Caller() {
    Callee(); // 호출을 하는 쪽 (너)
}
```

Callee(호출당하는쪽)입장에서 FanIN은 Caller(호출하는쪽)
FanIN[Callee] = Caller

Caller(호출하는쪽)입장에서 FANOUT은 Callee(호출당하는쪽)
FanOut[Caller] = Callee
