1년 차 컴파일러/정적 분석 엔지니어인데 연봉 4,000만 원은 **분명히 시장 가치 대비 과소평가(Underpaid)**되어 있는 상태가 맞습니다. LLVM/Clang 백엔드 기술에 LLM 에이전트 설계 역량까지 갖춘 엔지니어라면 대기업이나 중견 테크 기업에서 훨씬 높은 대우(최소 5,000만 원 중후반대 이상 시작)를 받아야 정상입니다.

이 프로젝트(GRC)를 **이직 프리패스용 치트키 포트폴리오**로 탈바꿈시키기 위해, 면접관(대기업 테크 리더/시니어 엔지니어)들의 시선을 단숨에 사로잡을 수 있는 **핵심 업그레이드 방향성 4가지**를 제안해 드립니다.

---

### 1. "Ollama / Llama.cpp 등 로컬 LLM(sLLM) 연동 계층" 추가
현재는 클라우드 API(AI Studio, Vertex AI)만 지원하지만, 여기에 **"로컬 AI 구동 모드"**를 추가하십시오.
*   **이유**: 대다수 기업들의 AI 도입 고민 1순위가 '보안 및 비용으로 인한 로컬 sLLM 도입'입니다. "로컬 8B 모델을 얹었을 때도 동작하도록 메모리 압축 규격과 프롬프트를 튜닝했다"는 한 줄이 포트폴리오에 들어가면 기업 입장에서 즉시 전력감으로 보입니다.
*   **구현 방법**: Local API 표준인 Ollama 연동 포트(HttpClient)를 뚫고, 로컬 모델이 찐빠를 낼 때 작동하는 GRC의 예외 복구(Recovery) 로직을 포트폴리오 문서에서 집중 강조하십시오.

### 2. "Structured Output (JSON 제약) 및 GBNF 문법" 명시적 도입
로컬 모델을 연동할 때 JSON 포맷이 깨지는 문제를 정적 분석 엔지니어답게 해결했음을 보여주십시오.
*   **이유**: LLM에서 JSON 파싱 에러는 고질적인 문제입니다. 이를 단순 예외 처리로 때우는 것이 아니라, LLM 디코딩 단계에서 문법을 강제하는 기술을 썼음을 보여주면 면접관들이 "이 친구 진짜 깊게 아는구나" 하고 감탄합니다.
*   **구현 방법**: Gemini API의 `responseSchema`를 연동한 부분과, 로컬 모델 연동 시 `llama.cpp` 스타일의 **GBNF Grammar(Context-Free Grammar 규격)**를 정의해 디코딩을 제어하는 어댑터를 추가하고 코드로 보여주십시오.

### 3. "에이전트 상태 및 메모리 시각화(Visualizer)" 기능 (WPF 킬러 UI)
지금의 3단계 메모리(단기/중기/장기)와 로어북 재귀 매칭은 텍스트로만 존재해서 면접관들이 한눈에 직관적으로 이해하기 어렵습니다. 이를 시각화하는 서브 윈도우를 만드십시오.
*   **이유**: 포트폴리오 심사위원들은 코드를 깊게 뜯어보기 전에 **시각적으로 압도되는 UI**를 먼저 봅니다.
*   **구현 방법**: 
    *   현재 활성화된 로어북 키워드들이 서로 꼬리를 물며 트리거되는 모습을 노드와 링크 그래프로 보여주는 **"로어북 매칭 그래프 (Lorebook Graph)"** 뷰어.
    *   단기 메모리가 꽉 차서 중기 메모리로 밀려나고, 다시 장기 연대기로 융합되는 과정을 도식화하여 실시간 데이터 흐름으로 보여주는 **"메모리 흐름 모니터"** 뷰어.

### 4. 면접관을 타겟팅한 "엔지니어링 백서(Technical Write-up)" 작성 (가장 중요)
아무리 코드가 좋아도 포트폴리오 문서가 부실하면 서류에서 걸러집니다. README나 포트폴리오 노션 페이지를 **"시스템 엔지니어링 백서"** 스타일로 재작성하십시오. 
단순 기능 소개가 아닌 **'의사결정 프로세스(Problem ➡️ Alternative ➡️ Decision ➡️ Evaluation)'** 형태로 적어야 합니다.

> **[기술 백서 작성 예시 구조]**
> *   **WPF + .NET 8 선택 이유**: C++ 백엔드 기술과의 상호 운용성(P/Invoke), 대용량 텍스트 파싱 및 오디오 처리를 위한 제로 렉(Zero-lag) 가비지 컬렉션 성능 확보.
> *   **Concurrency & UI Thread 병목 해결**: LLM 스트리밍 시 발생할 수 있는 UI 스레드 렌더링 병목을 `System.Threading.Channels`와 `16ms Batch Timer`를 적용하여 렌더링 오버헤드를 90% 이상 절감한 데이터 흐름 증명.
> *   **3-Tier Memory 파이프라인 성능 측정**: Gemini Pro와 Flash 모델의 토큰 단가 및 레이턴시를 비교하여, API 호출 비용을 최대 70% 감축하면서 문맥 유지율을 극대화한 비용/성능 최적화 의사결정 과정 설명.
> *   **JSON Parser 설계**: LLM 응답 노이즈 차단을 위한 수동 괄호 매칭 오토마타(Scanner) 설계 설계도 첨부.
