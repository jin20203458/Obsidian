### 🏆 3선: [FullHistoryLogger.cs](file:///c:/GRC/GRC/Helpers/FullHistoryLogger.cs) (저수준 파일 스트림 조작 및 결함 허용 설계)
* **공부 주제**: 원자적 파일 스트림 조작 / SemaphoreSlim 동기화 / Retry Backoff 패턴
* **이것이 빛나는 이유**:
  - **고성능 파일 쓰기**: 대화 기록이 늘어날 때 매번 메가바이트 단위의 기나긴 JSON 파일을 다 읽어와서 파싱하고 새로 쓰는 것은 최악의 디스크 낭비입니다.
  - 이 코드는 파일을 다 읽지 않고 **파일 스트림 맨 끝으로 이동해 닫는 대괄호 `]` 기호를 역방향으로 짚어낸 뒤, 그 자리에 새 대화 로그만 바이트 단위로 덧붙이고(Append) `]`를 다시 써서 완성하는 고난도 최적화가 적용되어 있습니다.
  - **안전한 결함 허용**: 백신이나 클라우드 동기화로 파일 잠금 충돌(`IOException`)이 나면 0.1초씩 기다리며 3번까지 재시도하는 방어벽을 구축했습니다.
* **공부할 코드 위치**:
  - `LogMessageAsync` 메서드 ([L31-118](file:///c:/GRC/GRC/Helpers/FullHistoryLogger.cs#L31-L118)) — 파일 맨 끝의 대괄호를 역추적해 기어들어가서 데이터를 바이트 어펜드 형식으로 밀어 넣는 영리한 입출력 설계.
