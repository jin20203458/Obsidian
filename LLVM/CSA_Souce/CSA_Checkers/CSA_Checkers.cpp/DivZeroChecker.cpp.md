> **0으로 나누기”** 오류를 탐지하는 정적 분석 체커

- ExprEngine(경로별 심볼릭 실행 엔진)에서  
    **나눗셈 연산**이 나올 때마다,  
    **분모 값이 0이 될 수 있는지** state(상태)를 추적
    
- 0이 될 수 있으면 **경고 리포트**
    
- 0이 확실하지 않으면(예를 들어, 조건 분기 등에서 불확실할 때)  
    → 일부 경로에서만 암묵적 경고 처리
    
- 실제 런타임 예외가 발생할 수 있는 잠재적 버그를 **개발 단계에서 미리 발견**

### 예시 
```cpp
int f(int x) {
    int z = x - 3;
    return 5 / z; // z가 0일 수 있다면 경고 발생!
}
```
