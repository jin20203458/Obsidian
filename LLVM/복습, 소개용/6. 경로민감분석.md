경로 민감 분석(Path-Sensitive Analysis)**은 프로그램의 **모든 가능한 실행 경로를 개별적으로 추적**하여 각 경로에서 변수의 상태, 제약 조건, 메모리 상태 등을 별도로 관리하는 정적 분석 기법입니다.
>  **Constraint Propagation**
## 기본 원리

- **경로별 상태 추적**: 각 실행 경로마다 독립적인 프로그램 상태(state)를 유지
    
- **제약 조건 누적**: 분기 조건들을 제약 조건으로 누적하여 각 경로의 유효성 검증
    
- **심볼릭 실행**: 구체적인 값 대신 심볼릭 값을 사용하여 모든 가능한 값에 대해 분석

## 경로 둔감 vs 경로 민감 분석

## 경로 둔감 분석 (Path-Insensitive)
```cpp
int x = 0;
if (condition) {
    x = 5;
} else {
    x = 10;
}
// 경로 비민감: x = {5, 10} (두 값 모두 가능)
```

## 경로 민감 분석 (Path-Sensitive)
```cpp
int x = 0;
if (condition) {
    x = 5;         // 경로 1: x = 5
} else {
    x = 10;        // 경로 2: x = 10
}
// 각 경로별로 정확한 값 추적
```
