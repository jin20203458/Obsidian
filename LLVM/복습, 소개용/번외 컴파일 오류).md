
# 컴파일 오류: 도대체 어디서 오는 걸까?

프로그래머라면 누구나 지겹도록 마주하는 빨간 줄, "컴파일 오류". 우리는 이 오류 메시지를 보고 코드를 수정하지만, 정작 이 오류가 컴파일러의 어느 단계에서, 왜 발생하는지는 잘 알지 못합니다. 컴파일 오류의 정체를 파헤쳐 보면, 컴파일러의 동작 원리를 더 깊이 이해할 수 있습니다.

## 컴파일 과정: 오류를 찾아내는 4단계 필터

컴파일러는 소스 코드를 기계어로 번역하는 과정에서 여러 단계를 거치며, 각 단계는 코드의 유효성을 검사하는 필터 역할을 합니다.

## 1단계: 어휘 분석 (Lexical Analysis)

- **역할**: 코드를 단어(토큰) 단위로 분해합니다.
    
- **발견하는 오류 (약 1%)**: 가장 기본적인 텍스트 오류입니다.
    
    - 닫히지 않은 따옴표: `const char* s = "hello;`
        
    - 잘못된 문자 포함: `int a = 10#;`
        
- **한계**: 코드의 '의미'나 '문법'은 전혀 보지 않고, 글자 그대로만 봅니다.
    

## 2단계: 구문 분석 (Syntax Analysis)

- **역할**: 단어(토큰)들이 언어의 문법 규칙에 맞게 배열되었는지 검사합니다.
    
- **발견하는 오류 (약 5%)**: 우리가 흔히 저지르는 문법 실수입니다.[namu](https://namu.wiki/w/%EC%BB%B4%ED%8C%8C%EC%9D%BC%20%EC%97%90%EB%9F%AC)​
    
    - 세미콜론(;) 누락
        
    - 괄호 `{}` 또는 `()` 불일치
        
    - 잘못된 키워드 사용 (`integer` 대신 `int`)
        
- **한계**: 문장의 구조는 보지만, 그 문장이 논리적으로 말이 되는지는 판단하지 못합니다. "사과가 철수를 먹는다"처럼 문법적으로는 완벽하지만 의미가 틀린 문장은 통과시킵니다.
    

## 3단계: 의미 분석 (Semantic Analysis) - **오류의 90%가 발생하는 곳**

- **역할**: 코드의 '의미'가 논리적으로 올바른지, C++ 언어의 모든 복잡한 규칙을 준수하는지 검사합니다.[poododang.tistory+1](https://poododang.tistory.com/entry/studyjava-1)​
    
- **발견하는 오류 (약 90%)**: 프로그래머가 마주하는 **대부분의 의미 있는 오류**가 여기서 발생합니다.
    
    - **타입 불일치**: `int a = "hello";` (`int`에 문자열 대입 불가)
        
    - **미선언 식별자**: `y = x;` (선언된 적 없는 `x` 사용)
        
    - **접근 제어 위반**: 외부에서 `private` 멤버에 접근
        
    - **함수 호출 오류**: `void foo(int);`를 `foo("text");`로 호출
        
- **특징**: 이 단계에서 비로소 컴파일러는 코드의 맥락을 이해하고, 문법적으로는 맞지만 논리적으로는 틀린 부분을 정확히 지적해 줍니다. Clang 같은 현대 컴파일러의 친절하고 상세한 오류 메시지는 대부분 이 "의미 분석기(Sema)"의 작품입니다.
    

## 4단계: 링킹 (Linking)

- **역할**: 컴파일된 여러 파일을 하나로 합쳐 최종 실행 파일을 만듭니다.
    
- **발견하는 오류 (약 4%)**: 개별 파일에서는 문제가 없지만, 합치는 과정에서 발생하는 오류입니다.
    
    - **정의되지 않은 참조 (Undefined Reference)**: 함수를 선언만 하고 어디에서도 구현하지 않았을 때.[woodforest.tistory](https://woodforest.tistory.com/173)​
        
    - **중복 정의 (Duplicate Symbol)**: 똑같은 이름의 함수가 여러 파일에 구현되어 있을 때.
        

## 결론: 오류 메시지는 컴파일러의 선물

컴파일 오류는 단순히 귀찮은 존재가 아닙니다. 코드가 실행되기도 전에 잠재적인 문제를 미리 알려주는 컴파일러의 중요한 피드백입니다. 특히 우리가 마주하는 오류의 대부분이 발생하는 **의미 분석 단계**는, 복잡한 C++의 규칙 속에서 우리가 저지르기 쉬운 논리적 실수를 잡아주는 가장 강력한 안전망 역할을 합니다.
