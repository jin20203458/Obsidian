
**CFG(Control Flow Graph)**는 프로그램의 **제어 흐름(분기, 반복 등)**을 그래프 구조로 나타낸 자료구조입니다. 각 노드는 코드의 실행 단위(보통 BasicBlock)이고, 엣지는 명령이 실행될 수 있는 경로(분기, 반복, 함수 호출 등)입니다.

- **노드(Node)**: 연속적으로 실행되는 명령 집합, 분기가 일어나기 전까지의 코드 블록 (BasicBlock)
    
- **엣지(Edge)**: 프로그램 실행 흐름에 따른 연결 (if, else, for, while 등의 분기 또는 반복)
    

## 2. Clang에서의 CFG 구성

Clang은 소스코드를 파싱한 뒤 AST를 기반으로 각 함수마다 **CFG를 생성**합니다.

- **FunctionDecl(AST)** → **CFG::buildCFG()** → **CFG 객체**
    
- CFG 내부는 여러 개의 BasicBlock(노드)로 구성되며, 제어 흐름에 따라 노드가 서로 연결됨.

```cpp
int main() {

    int x;
    if(x > 0) 
    {
        x++;
    } 
    else 
    {
        x --;
	}
    return 0;

```
