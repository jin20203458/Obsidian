

LLVM 정적 분석기(Static Analyzer)는 내부적으로 수십 가지의 `-analyzer-config` 옵션을 제공합니다. 회원님께서 만드신 C# 자동화 툴의 **"설정(Settings) 탭"**에 나중에 추가해 두면 아주 유용할, **핵심적이고 강력한 옵션**들을 카테고리별로 엄선해 보았습니다.

---

### 🚀 1. 분석 깊이 & 성능 조절 (Depth & Performance)

정적 분석기는 "얼마나 깊게(오래) 파고들 것인가"와 "얼마나 빨리 끝낼 것인가" 사이의 줄다리기입니다. 이 옵션들로 엔진의 체력을 조절할 수 있습니다.

- **`max-nodes`** (기본값: 보통 225,000)
    
    - **설명:** 엔진이 하나의 함수를 분석할 때 방문할 수 있는 **최대 노드(경로)의 수**입니다. 코드가 너무 복잡해서 경우의 수가 수백만 개로 폭발하면, 엔진은 이 수치에서 분석을 포기하고 다음 코드로 넘어갑니다.
        
    - **활용:** 버그를 더 꼼꼼하게 찾고 싶다면 `500000` 등으로 늘리고, 분석 속도를 높이고 싶다면 줄입니다.
        
    - **적용:** `-Xclang -analyzer-config -Xclang max-nodes=500000`
        
- **`max-inlinable-size`** (기본값: 보통 100)
    
    - **설명:** 다른 함수를 호출할 때, 그 함수의 코드 라인 수(또는 복잡도)가 이 수치보다 작을 때만 함수 내부로 파고들어 갑니다(Inlining). 너무 큰 함수를 파고들면 메모리가 터지기 때문입니다.
        
    - **활용:** 덩치가 큰 외부 함수 내부까지 깊게 분석하고 싶을 때 수치를 늘려줍니다.
        

### 🧠 2. 수학적 오탐지 제거 (Z3 SMT Solver)

LLVM 정적 분석기에 숨겨진 가장 강력하고 경이로운 기능 중 하나입니다. (단, LLVM 빌드 시 Z3가 포함되어 있어야 작동합니다.)

- **`crosscheck-with-z3`** (기본값: false)
    
    - **설명:** 엔진이 버그를 찾으면, 곧바로 보고하지 않고 **Z3 수학 증명기(SMT Solver)**에게 해당 코드를 방정식으로 만들어서 던집니다. "이 `if` 문들을 다 통과해서 여기에 도달하는 게 논리적(수학적)으로 진짜 가능해?" 라고 묻는 것입니다.
        
    - **활용:** 절대 일어날 수 없는 경로(Dead Code)에서 발견된 가짜 버그(False Positive)들을 수학적으로 증명해서 완벽하게 날려버립니다. 분석 시간은 좀 길어지지만 리포트의 신뢰도가 99.9%로 올라갑니다.
        
    - **적용:** `-Xclang -analyzer-config -Xclang crosscheck-with-z3=true`
        

### 🛡️ 3. 오지랖(FP Suppression) 제어 옵션들

아까 우리가 CTU를 위해 강제로 껐던 녀석들의 친구들입니다.

- **`suppress-inlined-defensive-checks`** (기본값: true)
    
    - 우리를 괴롭혔던 바로 그 옵션입니다. 인라인된 함수 내부의 방어 코드(Defensive Check)로 인한 버그를 무시합니다. (CTU 환경에서는 `false`로 끄는 것을 추천합니다.)
        
- **`suppress-null-return-paths`** (기본값: true)
    
    - 추적 결과 Null의 출처가 `return NULL;` 처럼 명시적인 반환값인 경우 경고를 무시합니다. (이것도 `false`로 끄는 것이 숨은 버그를 찾는 데 좋습니다.)
        
- **`suppress-c++-stdlib`** (기본값: true)
    
    - C++ 표준 라이브러리(STL) 내부 깊은 곳에서 발생하는 경고를 무시합니다. (STL 자체 버그보다는 분석기의 오해일 확률이 높으므로 켜두는 게 좋습니다.)
        

### 🕵️‍♂️ 4. 디버깅 & 로그 중계 (Visibility)

분석기가 대규모 프로젝트에서 프리징(멈춤)된 것 같은 답답함을 해소해 주는 옵션들입니다.

- **`analyzer-display-progress`** (기본값: false)
    
    - **설명:** 아까 썼던 `display-ctu-progress`는 AST 로드 상황만 보여주지만, 이 옵션을 켜면 엔진이 **"지금 정확히 어떤 파일의, 어떤 함수를 분석 중인지"** 실시간으로 콘솔에 중계해 줍니다.
        
    - **적용:** `-Xclang -analyzer-config -Xclang analyzer-display-progress=true`
        
    - **활용:** C# UI 쪽에 "현재 분석 중인 함수: `processData()`" 처럼 상태 바(Status Bar)를 띄워줄 때 아주 유용합니다.

### 💡 C# 툴 개발 팁 (UI 연동)

이 옵션들은 C# 자동화 툴의 핵심 경쟁력이 될 수 있습니다.

예를 들어 `ClangTidyRunnerService.cs`에 `AnalysisProfile` (분석 프로필) 같은 열거형(Enum)이나 클래스를 만드신 뒤, 사용자가 UI에서 아래처럼 선택하게 만들면 상용 툴 부럽지 않은 퀄리티가 나옵니다.

1. **🚀 Fast Mode (빠른 분석):** `max-nodes=75000`, CTU 끄기
    
2. **⚖️ Standard Mode (표준 분석):** 기본값, CTU 켜기
    
3. **🕵️ Deep Mode (심층 분석):** `max-nodes=500000`, `max-inlinable-size=300`, 억제기 끄기 (시간은 오래 걸리지만 영혼까지 털어서 버그를 찾아냄)
