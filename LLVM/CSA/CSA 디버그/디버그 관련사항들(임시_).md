## 🛠️ 클린 빌드 및 최종 설정 요약

### ✅ 완료된 클린 빌드 과정

- **LLVM/Clang 완전 빌드**: Debug 모드, Assertion 활성화
- **Checkers.td 정밀 수정**: 문제 체커들에 Hidden 속성 추가
- **디버그 체커 시스템 활성화**: Debug 패키지 Hidden 제거
- **체커 등록 시스템 정상화**: 의존성 검증 통과

### 🔧 핵심 수정사항

```cpp
// Checkers.td에서 수정된 부분들:
def Debug : Package<"debug">;  // Hidden 제거

// 문제 체커들에 Hidden 추가:
def ErrnoTesterChecker : Checker<"ErrnoTest">, Hidden;
def DebugContainerModeling : Checker<"DebugContainerModeling">, Hidden;  
def DebugIteratorModeling : Checker<"DebugIteratorModeling">, Hidden;
def StdCLibraryFunctionsTesterChecker : Checker<"StdCLibraryFunctionsTester">, Hidden;
```

## 🧠 기술적 이해도 획득

### LLVM Static Analyzer 내부 구조

- 체커 등록 시스템 동작 원리 파악
- Strong vs Weak Dependencies 이해
- Hidden 속성의 역할 및 사용자 노출 제어 방식 숙지

### Graph 구조 차이점 명확화

|그래프 종류|설명|
|---|---|
|**Call Graph**|함수 간 호출 관계 (Inter-procedural)|
|**CFG**|함수 내부 제어 흐름 (Intra-procedural)|
|**Exploded Graph**|CFG + 심볼릭 상태 (State-sensitive)|

### 디버그 체커 생태계 파악

- 모델링 체커 vs 진단 체커 구분
- 디버그/개발자 전용 체커 용도별 분류

## 🔍 핵심 기술적 발견사항

### 체커 등록 시스템의 까다로운 검증

```cpp
assert((DependencyIt->FullName.starts_with("test") ||
        DependencyIt->FullName.starts_with("example") || IsWeak ||
        DependencyIt->IsHidden) &&
       "Strong dependencies are modeling checkers, and as such "
       "non-user facing! Mark them hidden in Checkers.td!");
```

- Strong dependencies는 반드시 Hidden이어야 함
- 사용자 혼란 방지를 위한 설계
- 개발 목적에 따라 우회 가능

### EvalAssume 체커 개발 기반 구축

- Exploded Graph 구조 완전 이해
- 심볼릭 실행 및 분기 조건 분석 메커니즘 파악
- 그래프 확장 제어 아이디어 확보
