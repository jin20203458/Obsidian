
## TypeSourceInfo란?

**TypeSourceInfo**는 Clang의 AST(추상 구문 트리)에서  
“어떤 타입(type)이 소스코드에 **어떻게 적혀 있는지**에 대한 위치 정보와 세부 구문 구조”를 함께 담는 객체입니다.

## 1. 왜 필요한가?

- 일반적인 타입은(예: `int`, `double`, `std::vector<int>`) 타입 그 자체만 저장하면 충분해 보이지만,
    
- 실제 소스코드에는 타입이 다양한 문법(예: 포인터, 참조, 템플릿, 배열, traling return type 등)으로 복합적으로 작성될 수 있습니다.
    
- 코드 리팩토링, 진단 메시지, 코드 자동화 수정 등에서 **타입이 정확히 “소스에서” 어떻게 작성됐는지**를 알아야 할 때가 많습니다.
    

## 2. 주요 역할/구성

- **타입 자체(일반 Type) 정보** + **타입의 소스 위치(어디에 쓰였는지)** + **해당 타입 구문의 실제 구조**
    
- 내부적으로는 `TypeLoc`이라는 객체로 감싸서 타입의 트리 구조 정보(포인터, 배열, 래퍼 등)를 제공받을 수 있습니다.
    

## 3. 대표적 사용 사례

- 함수/변수/필드 선언에서, 타입 뒤에 붙는 다양한 구문 정보를 파악하고 싶을 때  
    (예: 함수의 trailing return type(ex: `auto func() -> int`) 유무, 템플릿 인자 위치 등)
    
- 자동 코드 수정(fix-it)이나, 진단 메시지에서 **정확한 소스코드 범위**를 잡아주고 싶을 때
    
- 변수 선언에서, 타입 토큰이 보이는 정확한 “소스 범위”를 출력 영역으로 써야 할 때
    

## 4. 실제 예시

```cpp
int *ptr;
std::vector<int> v;
auto func() -> int;
```

위 코드에서 각 선언의 타입에 대해 목록을 구축하려면, 단순 타입 정보만으로는 소스코드 상에서 `*`, `<int>`, `-> int` 등 각종 부가 구문 정보를 완전히 복원하기 어렵습니다.  
→ 이럴 때 TypeSourceInfo를 통해 **타입이 등장한 실제 소스 지점, 내부 구조, 토큰 순서**까지 모두 알아낼 수 있습니다.

## 주요 멤버 및 활용

- **getTypeLoc()**
    
    - TypeSourceInfo에서 TypeLoc 객체를 얻을 수 있음
        
    - TypeLoc은 타입이 소스에서 어떻게 구체적으로 작성됐는지에 대한 node-by-node(트리 형태) 구문 정보를 갖습니다.
        
- **getType()/getTypeLoc() → getBeginLoc()/getEndLoc()**
    
    - 타입이 소스 상에서 어디서 시작·끝나는지 위치 정보(Clang SourceLocation)를 얻을 수 있음.
        

## 비유

**Type만 있으면 “이 변수는 int다”까지만 알지만,  
TypeSourceInfo가 있으면 “이 변수의 int라는 말이 소스코드 몇 번째 줄, 어디 좌표에서 시작되고,  
뒤에 포인터(*)가 붙었는지, 배열인지, 템플릿 인자인지 등 모든 타이핑 정보와 위치까지 통째로 알 수 있다”**  
라고 생각하시면 됩니다.

## 요약 표

| 항목           | 설명                                                    |
| ------------ | ----------------------------------------------------- |
| **정의**       | 타입의 소스코드상 구문/위치/트리구조 정보를 담은 Clang AST객체               |
| **주요 역할**    | 타입 위치(시작/끝), 트리 구조, 서브 구문(포인터, 참조 등) 분석               |
| **활용 예**     | 자동 코드 수정, 진단 메시지, trailing return 등 복합 타입 구문 판별       |
| **주요 인터페이스** | getType(), getTypeLoc(), getBeginLoc(), getEndLoc() 등 |
