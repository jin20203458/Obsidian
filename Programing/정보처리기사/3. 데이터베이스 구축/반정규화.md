### 📂 반정규화란?

- 이미 정규화된 테이블을 **일부러 정규화 이전 상태로 되돌리는 것**
    
- 목적은 **성능 향상**: 조회 속도를 빠르게 하고, 조인 연산을 줄이기 위해 사용함
    

### 🚧 왜 반정규화를 할까?

|상황|이유|
|---|---|
|**JOIN이 너무 많아 성능 저하**|정규화된 테이블이 많을수록 조회 시 JOIN 연산이 늘어나 속도가 느려짐|
|**실시간 성능이 중요**|정규화 구조는 무결성은 좋지만 실시간 처리에 부적합할 수 있음|
|**조회가 많고 수정은 적을 때**|중복 데이터를 허용해도 일관성 유지에 큰 문제가 없을 경우|

### 🔁 반정규화 기법

|기법|설명|
|---|---|
|**중복 속성 추가**|다른 테이블에 있는 데이터를 복사해 중복 저장|
|**테이블 통합**|자주 조인되는 두 테이블을 하나로 합침|
|**파생 속성 추가**|계산 결과를 미리 테이블에 저장 (예: 합계, 평균 등)|
|**중간 테이블 생성**|다대다 관계를 단순화하거나 조회용 서브 테이블을 따로 만듦|

### ⚠️ 단점도 있어요!

- **데이터 중복 증가** → 무결성 관리가 어려워질 수 있음
    
- **수정 작업 증가 시 오류 가능성** → 중복된 데이터가 일관되게 수정돼야 함
    
- **업데이트 비용**이 커질 수 있음
    

### ✅ 시험 포인트 요약

- 반정규화는 **조회 성능 향상을 위해** 정규화된 테이블을 **의도적으로 분해하지 않고 되돌리는 것**
    
- 데이터 중복을 허용하지만 **성능을 위해 타협**하는 전략
    
- 실무에서 읽기 성능이 중요한 경우 자주 사용됨
