
### [연산자-가] 시퀀스 포인트 내 중복 변경 금지
- checkerName: SequencingRulesCheck
- category: Operator, Undefined Behavior
- severity: CRITICAL
- summary: "하나의 시퀀스 포인트(예: 세미콜론 이전) 내에서 동일한 변수를 두 번 이상 수정하면 안 됩니다."
- violation: `x = x++ * x++;`
- correction: `y = x * x; x += 2;`

### [연산자-나] 0으로 나누기 방지
- checkerName: DivZeroChecker, NonZeroDivisorGuardCheck
- category: Operator, Runtime Error
- severity: CRITICAL
- summary: "나눗셈 연산(/, %) 수행 시, 나누는 값이 0이 아닌지 반드시 확인해야 합니다."
- violation: `r = x / n;`
- correction: `if (n != 0) { r = x / n; }`

### [연산자-다] 시퀀스 포인트 내 변경 및 접근 금지
- checkerName: SequencingRulesCheck
- category: Operator, Undefined Behavior
- severity: CRITICAL
- summary: "하나의 시퀀스 포인트 내에서 변수 값을 변경함과 동시에 다른 곳에서 읽으면 안 됩니다."
- violation: `r = (2 * index) + index++;`
- correction: `r = (3 * index); index++;`

### [연산자-라] 잘못된 시프트 연산 금지
- checkerName: BitwiseShiftChecker, ShiftAmountRangeCheck
- category: Operator, Bitwise
- severity: ERROR
- summary: "시프트 연산(<<, >>) 시, 음수 값이나 데이터 타입의 비트 수를 초과하는 값으로 시프트할 수 없습니다."
- violation: `uia = uia << -2;`, `uia = uia << 40;` (32[translate:비트 기준])
- correction: `uia = uia >> 2;`

### [연산자-마] unsigned 타입에 단항 음수 연산 금지
- checkerName: UnsignedMinusAssignmentCheck
- category: Operator, Type Conversion
- severity: MAJOR
- summary: "unsigned 타입 변수에 단항 마이너스(-) 연산자를 적용하면 의도치 않은 큰 양수 값으로 변경될 수 있습니다."
- violation: `unsigned char i; j = -i;`
- correction: `signed char i; j = -i;`

### [연산자-바] sizeof 인자에 Side Effect 금지
- checkerName: UnevaluatedSideEffectCheck
- category: Operator
- severity: WARNING
- summary: "sizeof 연산자의 피연산자로는 증가(++), 감소(--) 등 Side Effect가 있는 표현식을 사용하지 않습니다 (실행되지 않음)."
- violation: `un = sizeof(++y);`
- correction: `y++; un = sizeof(y);`

### [연산자-사] Boolean 값에 논리 연산자만 사용
- checkerName: BoolLogicOnlyOpsCheck
- category: Operator, Boolean
- severity: WARNING
- summary: "Boolean(참/거짓) 값을 다룰 때는 비트(~, |)나 산술(+, -) 연산자 대신 논리 연산자(&&, ||, !)만을 사용해야 합니다."
- violation: `x = ~((a > b) && (b > 10));`
- correction: `if (!((a > b) && (b > 10))) { x = 1; }`

### [연산자-아] 조건문 내 대입 연산자 사용 금지
- checkerName: NoAssignmentInConditionCheck
- category: Operator, Condition
- severity: ERROR
- summary: "조건문(if, while 등) 안에서 비교 연산자(==) 대신 대입 연산자(=)를 사용하는 실수를 방지해야 합니다."
- violation: `if (var2 = var1)`
- correction: `if (var2 == var1)`

### [연산자-자] signed 타입에 비트 연산 주의
- checkerName: SignedBitwiseSafetyCheck, BitwiseShiftChecker
- category: Operator, Bitwise
- severity: MAJOR
- summary: "signed 타입 변수에 비트 연산을 적용하면 부호 비트(sign bit)가 변경되어 예기치 않은 결과가 발생할 수 있습니다."
- violation: `long sl; slr = sl << 4;`
- correction: `unsigned int sl; slr = sl << 4;`
