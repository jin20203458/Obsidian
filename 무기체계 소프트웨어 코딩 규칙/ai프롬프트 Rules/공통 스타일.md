
### [스타일-나] Switch Label 전 코드 금지
- checkerName: SwitchStyleCheck
- category: Control Flow
- severity: ERROR
- summary: "switch문에서 case/default 레이블 앞에 실행 코드가 올 수 없습니다."
- violation: `switch(x) { stmt; case 1: ... }`
- correction: `switch(x) { case 1: ... }`

### [스타일-다] 명시적 타입 선언
- checkerName: NoAutoTypeCheck
- category: Type Safety
- severity: WARNING
- summary: "변수/함수 선언 시 auto 대신 명시적 타입을 사용해야 합니다."
- violation: `auto x = 10;`, `auto f() { return 1; }`
- correction: `int x = 10;`, `int f() { return 1; }`

### [스타일-라] 의미 없는 구문 금지
- checkerName: NoMeaninglessExprCheck
- category: Performance
- severity: WARNING
- summary: "결과를 사용하지 않는 비교 연산 등 Side Effect가 없는 구문은 제거해야 합니다."
- violation: `x == y;`
- correction: `x = y;` 또는 구문 삭제

### [스타일-마] 반환 타입 일치
- checkerName: ReturnTypeCheck
- category: Type Safety
- severity: ERROR
- summary: "함수 선언부의 반환 타입과 return문의 실제 반환 값 타입은 일치해야 합니다."
- violation: `int f() { return; }`, `void f() { return 1; }`
- correction: `int f() { return 0; }`, `void f() { return; }`

### [스타일-바] 외부 함수 사전 선언
- checkerName: ExternFunctionDeclarationCheck
- category: Declaration
- severity: ERROR
- summary: "외부 함수는 호출 전에 extern으로 선언해야 합니다."
- violation: `func(0);` (선언 없이 호출)
- correction: `extern int func(int); func(0);`

### [스타일-사] 매크로 사전 정의
- checkerName: MacroDefinedBeforeUseCheck
- category: Preprocessor
- severity: WARNING
- summary: "#if/#elif에서 사용하는 매크로는 사용 전에 반드시 정의되어야 합니다."
- violation: `#if (UNDEFINED_MACRO == 1)`
- correction: `#ifndef M\n#define M 0\n#endif\n#if(M==1)`

### [스타일-아] goto 사용 금지
- checkerName: NoGotoCheck
- category: Control Flow
- severity: MAJOR
- summary: "goto문은 코드 흐름을 복잡하게 만들므로 사용을 금지합니다."
- violation: `goto my_label;`
- correction: `while/for/if` 등 구조적 제어문 사용

### [스타일-자] 단일 Exit Point
- checkerName: SingleExitCheck
- category: Control Flow
- severity: INFO
- summary: "함수는 여러 개의 return문을 갖지 않고, 단 하나의 지점에서만 종료해야 합니다."
- violation: `if(c) return 1; else return 0;`
- correction: `int r; if(c) r=1; else r=0; return r;`

### [스타일-차] switch문 default 필수
- checkerName: SwitchStyleCheck
- category: Control Flow
- severity: WARNING
- summary: "모든 switch문은 예측하지 못한 값을 처리하기 위해 default 레이블을 포함해야 합니다."
- violation: `switch(x) { case 1: ... }`
- correction: `switch(x) { case 1: ...; default: ...; }`

### [스타일-카] 한 줄에 한 명령문
- checkerName: MultiStatementPerLineCheck
- category: Style
- severity: INFO
- summary: "가독성을 위해 한 줄에는 하나의 명령문만 작성해야 합니다."
- violation: `x = 1; y = 2;`
- correction: `x = 1;\ny = 2;`

### [스타일-타] if-else if-else 구조
- checkerName: IfStyleCheck
- category: Control Flow
- severity: WARNING
- summary: "if-else if 체인은 모든 경우를 처리하기 위해 마지막에 else문을 포함해야 합니다."
- violation: `if(c1) ... else if(c2) ...`
- correction: `if(c1) ... else if(c2) ... else ...`
