

CFG 활용 체커들은 해당사항을 모두 확인할 필요가 있음

- **현상**: `if (d != 0)`라는 명백히 안전한 코드에서 잘못된 경고(False Positive)가 발생했습니다.
    
- **원인 발견**: 디버깅 결과, 데이터 흐름 분석 엔진(`isDefinitelyNonZeroAtUse`)이 나눗셈 연산이 포함된 CFG 블록(`UseBlock`)을 찾는 데 실패하여 시작과 동시에 조기 종료되는 것을 발견했습니다. Matcher가 찾은 `BinaryOperator`(`100 / d`)는 CFG가 가리키는 `DeclStmt`(`int result = ...`)의 **자손 노드**였기 때문에, 단순한 포인터/위치 비교로는 절대 찾을 수 없었습니다.
- **해결책**: 재귀적으로 자식 노드를 탐색하는 **`stmtContains` 헬퍼 함수를 도입**했습니다. 이를 통해 "CFG의 문장이 우리가 찾던 문장을 포함하는가?"라는 더 근본적인 질문을 던져, 복잡한 AST 구조 속에서도 `UseBlock`을 100% 찾아낼 수 있게 되었습니다.
