# 3. Cppcheck 규칙 검출 정량적 통계 데이터

본 문서는 Cppcheck v2.21.0이 DAPA 무기체계 소프트웨어 코딩 규칙 66종을 대상으로 수행한 정적 분석 성능을 **수치화(Metrics)하여 정리한 통계 자료**입니다.
이 데이터는 품질 관리 지표, 타 도구(Clang-Tidy) 도입 당위성 산출 등에 활용할 수 있습니다.

## 3.1. 전역 지표 요약 (Global Metrics)

| 지표 (Metrics) | 수치 (Value) | 비고 |
| :--- | :---: | :--- |
| **총 대상 규칙 (Total Rules)** | 66 | DAPA 기준 |
| **분석 제외 (Excluded)** | 1 | 주석 비율 검사 등 정적 분석 범위를 벗어난 규칙 |
| **실제 분석 대상 (Valid Total)** | 65 | 검증 모수에 해당 |
| | | |
| **✅ 정탐 (True Positives)** | **29** | 규칙 의도에 정확히 부합하는 결함 검출 건수 |
| **❌ 미탐 및 파생 경고 (Missed)** | **36** | 아예 탐지 못했거나, 엉뚱한 부수적 경고만 낸 건수 |
| ↳ *완전 미탐 (Silent / False Negative)* | *17* | 아무런 경고도 출력되지 않은 건수 |
| ↳ *파생 경고 (Collateral Only)* | *19* | `unreadVariable` 등 결함과 무관한 경고만 띄운 건수 |
| | | |
| **💡 실제 결함 검출률 (Detection Rate)** | **44.6%** | `(정탐 / 실제 분석 대상) * 100` |
| **📉 미탐률 (Miss Rate)** | **55.4%** | `(미탐 / 실제 분석 대상) * 100` |

## 3.2. 룰셋 카테고리별 검출 정량 데이터 (Category Metrics)

| 카테고리 | 전체 항목 | 검출(TP) | 미탐/파생(FN) | 탐지율(%) | 강약점 분석 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| Common_Style | 11 | 2 | 9 | **18.2%** | 문법적 스타일에 관련된 규칙으로, AST 분석 한계로 인해 탐지율 저조 (18.2%) |
| Common_Init | 3 | 2 | 1 | **66.7%** | 단순 변수 초기화는 잘 잡으나 배열 문자열 초기화 등 세부사항 미탐 |
| Common_Ident | 4 | 1 | 3 | **25.0%** | 링키지 및 식별자 범위 규칙에 약함 (Scope 제외 미탐) |
| Common_Cond | 5 | 2 | 3 | **40.0%** | 단순 조건식 논리 오류는 탐지하나 Float 비교, Switch 구조는 미탐 |
| Common_Conv | 8 | 2 | 6 | **25.0%** | 기본 타입 검증은 약하나 printf/scanf 인수 매칭은 강력함 |

| Common_Expr | 9 | 5 | 4 | **55.6%** | 기본 연산 논리(Shift, Order)는 탐지하나 논리 연산자 엄격성 부족 |
| C_Specific | 5 | 1 | 4 | **20.0%** | C 표준 구문 및 특정 함수 사이즈 연산에 대한 세밀함 부족 |
| CPP_Specific | 15 | 10 | 5 | **66.7%** | new/delete, Use After Free는 매우 강력하나, 상속 및 캐스팅 제약(스타일)에 약함 |

## 3.3. 결론적 정량 평가
위 정량 데이터를 바탕으로, Cppcheck는 포인터, 배열 메모리 안전성(탐지율 80%), 연산자 논리 및 C++ 생명주기 관련 검증(탐지율 60% 내외)에서는 강력한 성능을 지표로 입증하고 있습니다.
반면, **공통 코딩 스타일(탐지율 0%), 식별자 링키지, 형변환 제약 등 컴파일러 레벨의 제약**에서는 철저히 무력함(미탐률 100%)을 데이터로 보여줍니다.
결과적으로, 전체 탐지율 44.6%를 100%로 끌어올리기 위한 **정량적 솔루션으로 Clang-Tidy(ArqaStatic)의 병행 도입 체계가 필수불가결함**을 증명합니다.
