### **1) CWE-119: 메모리 버퍼 경계 내 연산 제한 부재**

[[MemoryBufferBoundsCheck]]

### ❌ 잘못된 예


```c
void update_array(int index, int value) {
    int buffer[10];
    buffer[index] = value; // ❌ 경계 검사 없음 (Buffer Overflow/Underflow)
}
```

#### 문제점

- `index`가 0보다 작거나 9보다 큰 경우, 할당되지 않은 메모리 영역에 접근하여 데이터 손상 및 프로그램 크래시를 유발합니다.
    

### ✅ 바른 예


```c
void update_array(int index, int value) {
    int buffer[10];
    // ✅ 인덱스 범위(0 ~ 9)를 명시적으로 확인
    if (index >= 0 && index < 10) {
        buffer[index] = value;
    }
}
```

### **2) CWE-170: 부적절한 널(Null) 종료**

[[NullTerminationCheck]]

### ❌ 잘못된 예


```c
void copy_data(char *src) {
    char dest[16];
    strncpy(dest, src, 16); // ❌ src가 16자 이상이면 dest에 '\0'이 없음
    printf("%s\n", dest);   // ❌ NULL 문자 없이 출력 시 메모리 오버런 발생
}
```

#### 문제점

- `strncpy`는 버퍼가 가득 차면 널 종료 문자를 삽입하지 않습니다. 이를 인지하지 못하고 출력하거나 사용할 경우 보안 취약점이 됩니다.
    

### ✅ 바른 예


```c
void copy_data(char *src) {
    char dest[16];
    strncpy(dest, src, sizeof(dest) - 1);
    dest[sizeof(dest) - 1] = '\0'; // ✅ 마지막 바이트에 명시적으로 NULL 삽입
}
```

### **3) CWE-190: 정수 오버플로우**

[[IntegerOverflowCheck]]

### ❌ 잘못된 예


```c
void process(int size) {
    int total_size = size + 100; // ❌ size가 INT_MAX 근처일 경우 오버플로우 발생
    char *buf = (char *)malloc(total_size);
}
```

#### 문제점

- 연산 결과가 데이터 타입의 범위를 넘어서면 의도치 않은 작은 값이나 음수가 되어, 부적절한 메모리 할당으로 이어집니다.
    

### ✅ 바른 예


```c
#include <limits.h>
void process(int size) {
    if (size > INT_MAX - 100) return; // ✅ 연산 전 오버플로우 가능성 체크
    int total_size = size + 100;
    char *buf = (char *)malloc(total_size);
}
```

### **4) CWE-400: 제어되지 않는 자원 소비**

[[ResourceConsumptionCheck]]

### ❌ 잘못된 예


```c
void handle_request(int count) {
    char *data = (char *)malloc(count * 1024); // ❌ 외부 입력값 'count'에 대한 제한 없음
    // ...
}
```

#### 문제점

- 공격자가 매우 큰 `count` 값을 전달하면 시스템 메모리가 고갈되어 서비스 거부(DoS) 상태가 됩니다.
    

### ✅ 바른 예


```c
#define MAX_ALLOWED 1024
void handle_request(int count) {
    if (count <= 0 || count > MAX_ALLOWED) return; // ✅ 최대 자원 사용량 제한
    char *data = (char *)malloc(count * 1024);
}
```

### **5) CWE-404: 부적절한 자원 해제**

[[ResourceReleaseCheck]]

### ❌ 잘못된 예


```c
void write_log(char *msg) {
    FILE *fp = fopen("log.txt", "a");
    if (msg == NULL) return; // ❌ 함수 종료 전 fp를 닫지 않음 (Resource Leak)
    fprintf(fp, "%s", msg);
    fclose(fp);
}
```

#### 문제점

- 특정 조건에서 `fclose`를 거치지 않고 종료되면 파일 핸들 누수가 발생하며, 반복될 경우 시스템이 더 이상 파일을 열 수 없게 됩니다.
    

### ✅ 바른 예


```c
void write_log(char *msg) {
    FILE *fp = fopen("log.txt", "a");
    if (fp == NULL) return;
    if (msg != NULL) {
        fprintf(fp, "%s", msg);
    }
    fclose(fp); // ✅ 모든 경로에서 자원이 해제되도록 보장
}
```

### **6) CWE-415: 이중 해제**

[[DoubleFreeCheck]]

### ❌ 잘못된 예


```c
void cleanup(char *ptr) {
    free(ptr);
    // ... logic ...
    free(ptr); // ❌ 이미 해제된 메모리를 다시 해제
}
```

#### 문제점

- 동일한 메모리 주소를 두 번 해제하면 힙(Heap) 구조가 파괴되어 임의 코드 실행이나 크래시가 발생할 수 있습니다.
    

### ✅ 바른 예


```c
void cleanup(char *ptr) {
    if (ptr != NULL) {
        free(ptr);
        ptr = NULL; // ✅ 해제 후 즉시 NULL 대입으로 재해제 방지
    }
}
```

### **7) CWE-416: 해제된 메모리 사용**

[[UseAfterFreeCheck]]

### ❌ 잘못된 예


```c
void uaf_example() {
    char *buf = (char *)malloc(10);
    free(buf);
    sprintf(buf, "error"); // ❌ 해제된 메모리(Dangling Pointer)에 접근
}
```

#### 문제점

- 이미 운영체제에 반환된 메모리 영역을 읽거나 쓰면 예측 불가능한 동작이 발생합니다.
    

### ✅ 바른 예


```c
void uaf_example() {
    char *buf = (char *)malloc(10);
    if (buf) {
        // ... 사용 ...
        free(buf);
        buf = NULL; // ✅ 다시 참조하지 못하도록 초기화
    }
}
```

```c
void check_status() {
```

```c
void check_status() {

    if (status == 1) {
        // ...
    }
}
```

### **9) CWE-467: 포인터 타입에 대한 sizeof() 사용**

[[SizeofPointerCheck]]
[공용]
### ❌ 잘못된 예


```c
void init_data(int *arr) {
    memset(arr, 0, sizeof(arr)); // ❌ arr 포인터 자체의 크기(4/8 bytes)만 초기화됨
}
```

#### 문제점

- 배열 전체를 초기화하려는 의도와 달리, 포인터 변수의 크기만큼만 처리되어 데이터 오염이나 버그가 발생합니다.
    

### ✅ 바른 예


```c
void init_data(int *arr, size_t count) {
    // ✅ 실제 요소의 개수와 타입 크기를 곱해서 계산
    memset(arr, 0, count * sizeof(int)); 
}
```

### **10) CWE-469: 크기 결정을 위한 포인터 뺄셈의 오용**

[[PointerSubtractionCheck]]

### ❌ 잘못된 예


```c
void diff_ptrs() {
    int buf1[10], buf2[10];
    int diff = &buf2[5] - &buf1[2]; // ❌ 서로 다른 배열 간의 뺄셈 (Undefined Behavior)
}
```

#### 문제점

- 서로 다른 메모리 블록에 할당된 포인터 간의 뺄셈 결과는 정의되지 않았으며, 잘못된 크기 계산으로 이어집니다.
    

### ✅ 바른 예


```c
void diff_ptrs() {
    int buf1[10];
    int diff = &buf1[5] - &buf1[2]; // ✅ 동일한 배열/블록 내에서만 뺄셈 수행
}
```

### **11) CWE-476: 널(NULL) 포인터 역참조**

[[NullPointerDereferenceCheck]]

### ❌ 잘못된 예


```c
void print_val(int *p) {
```

```c
void print_val(int *p) {

        printf("%d", *p);
    }
}
```

### **12) CWE-562: 스택 변수 주소 반환**

```c
char* get_temp_path() {
```

```c
void get_temp_path(char *out_path, size_t size) {
```

### **13) CWE-587: 포인터에 고정 주소 할당**

[공용]
[[FixedAddressAssignmentCheck]]

### ❌ 잘못된 예


```c
void write_to_hardware() {
    int *status_ptr = (int *)0x00001234; // ❌ 절대 메모리 주소 하드코딩
    *status_ptr = 1;
}
```

#### 문제점

- 주소값이 고정되면 다른 시스템에서 동작하지 않으며(이식성 결여), 보안상 공격자에게 고정된 공격 지점을 제공하게 됩니다.
    

### ✅ 바른 예


```c
int system_status; 
void write_to_hardware() {
    int *status_ptr = &system_status; // ✅ 논리적으로 할당된 메모리 주소 사용
    *status_ptr = 1;
}
```
