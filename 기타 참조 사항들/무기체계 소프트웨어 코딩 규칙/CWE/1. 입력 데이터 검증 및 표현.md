### **1) CWE-22: 경로 조작 및 자원 삽입 (Path Traversal)**


**규칙: 외부 입력값에서 상위 디렉터리 접근 문자(.. 등)를 필터링하거나, 허용된 경로 내의 파일만 접근하도록 제한해야 한다.**

#### ❌ 잘못된 예


```c
#include <stdio.h>

void read_user_file(char *user_input) {
    char path[100];
    sprintf(path, "/var/www/uploads/%s", user_input); // ❌ 경로 조작 취약점
    FILE *fp = fopen(path, "r");
    // ... 파일 읽기 수행
}
```

**문제점**

- `user_input`에 `../../etc/passwd`와 같은 값이 들어오면 의도치 않은 시스템 파일에 접근하게 됨.
- 외부 입력이 파일 경로 생성에 직접적으로 사용되어 접근 통제를 우회함.


#### ✅ 바른 예


```c
#include <string.h>
#include <stdio.h>

void read_user_file(char *user_input) {
    // ✅ ".." 문자열 포함 여부 검사 및 입력값 제한
    if (strstr(user_input, "..") != NULL) {
        return; 
    }
    char path[100];
    snprintf(path, sizeof(path), "/var/www/uploads/%s", user_input);
    FILE *fp = fopen(path, "r");
    // ...
}
```

### **2) CWE-59: 링크 따라가기 (Link Following)**

**규칙: 파일을 열기 전 해당 파일이 심볼릭 링크인지 확인하거나, 링크를 따라가지 않는 옵션을 사용해야 한다.**

#### ❌ 잘못된 예


```c
#include <stdio.h>

void write_config(char *filename) {
    FILE *fp = fopen(filename, "w"); // ❌ 심볼릭 링크 검사 없음
    fprintf(fp, "new configuration");
    fclose(fp);
}
```

**문제점**

- 공격자가 중요 시스템 파일(예: `/etc/shadow`)에 대한 심볼릭 링크를 `filename`으로 생성해두면, 프로그램이 엉뚱한 중요 파일을 덮어쓰게 됨.
    

#### ✅ 바른 예


```c
#include <fcntl.h>
#include <unistd.h>

void write_config(char *filename) {
    // ✅ O_NOFOLLOW 플래그를 사용하여 심볼릭 링크인 경우 열지 않음
    int fd = open(filename, O_WRONLY | O_CREAT | O_NOFOLLOW, 0644);
    if (fd == -1) return;
    write(fd, "new configuration", 17);
    close(fd);
}
```

### **3) CWE-78: 운영체제 명령어 삽입 (OS Command Injection)**


**규칙: 외부 입력값을 OS 명령어로 전달하지 않아야 하며, 반드시 필요한 경우 허용 목록(Whitelist) 방식으로 검증해야 한다.**

#### ❌ 잘못된 예


```c
#include <stdlib.h>
#include <stdio.h>

void run_ping(char *ip_addr) {
    char cmd[100];
    sprintf(cmd, "ping -c 4 %s", ip_addr); // ❌ 외부 입력값이 명령어의 일부가 됨
    system(cmd);
}
```

**문제점**

- `ip_addr`에 `127.0.0.1; rm -rf /`와 같은 값이 들어오면 악의적인 명령어가 실행될 수 있음.
    

#### ✅ 바른 예


```c
#include <unistd.h>

void run_ping(char *ip_addr) {
    // ✅ system() 대신 exec계열 함수를 인자값으로 분리하여 사용
    char *args[] = {"/bin/ping", "-c", "4", ip_addr, NULL};
    execv("/bin/ping", args);
}
```

### **4) CWE-89: SQL 삽입 (SQL Injection)**


**규칙: SQL 쿼리 생성 시 외부 입력값을 직접 연결하지 말고, Prepared Statement(파라미터화된 쿼리)를 사용해야 한다.**

#### ❌ 잘못된 예


```c
// ❌ 문자열 결합을 통한 쿼리 생성
sprintf(query, "SELECT * FROM users WHERE id = '%s'", user_id);
db_query(query);
```

**문제점**

- `user_id`에 `' OR '1'='1`을 입력하면 전체 사용자 정보가 유출됨.
    

#### ✅ 바른 예


```c
// ✅ 파라미터화된 쿼리 사용 (예시 프로토콜)
char *query = "SELECT * FROM users WHERE id = ?";
db_prepare(query);
db_bind_param(1, user_id);
db_execute();
```

### **5) CWE-99: 자원 식별자의 부적절한 제어 (Resource Injection)**


**규칙: 포트 번호, 소켓 등 시스템 자원 식별자에 외부 입력값을 직접 사용하지 말고 유효 범위를 검증해야 한다.**

#### ❌ 잘못된 예


```c
void connect_service(int port) {
    // ❌ 포트 번호를 외부에서 전달받은 그대로 사용
    serv_addr.sin_port = htons(port); 
    connect(sock, (struct sockaddr *)&serv_addr, sizeof(serv_addr));
}
```

**문제점**

- 공격자가 의도하지 않은 포트로 연결을 유도하여 서비스 거부(DoS)를 유발하거나 데이터 유출 경로로 활용할 수 있음.
    

#### ✅ 바른 예


```c
void connect_service(int port) {
    // ✅ 허용된 포트 범위 내에 있는지 검증
    if (port < 1024 || port > 65535) return;
    serv_addr.sin_port = htons(port);
    // ...
}
```

### **6) CWE-134: 포맷 스트링 삽입 (Format String Injection)**


**규칙: `printf`, `sprintf` 등의 함수 사용 시 외부 입력값을 포맷 문자열 인자로 직접 전달하지 않아야 한다.**

#### ❌ 잘못된 예


```c
#include <stdio.h>

void log_message(char *user_msg) {
    printf(user_msg); // ❌ 외부 입력이 포맷 스트링으로 사용됨
}
```

**문제점**

- `user_msg`에 `%x %x %n` 등이 포함될 경우 스택 메모리 정보가 유출되거나 특정 메모리 주소의 값이 변조될 수 있음.
    

#### ✅ 바른 예


```c
#include <stdio.h>

void log_message(char *user_msg) {
    printf("%s", user_msg); // ✅ 포맷 스트링 상수를 지정하고 입력값은 인자로 전달
}
```
