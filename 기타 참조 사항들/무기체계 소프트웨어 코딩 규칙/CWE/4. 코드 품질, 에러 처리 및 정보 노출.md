### **1) CWE-209: 오류 메시지를 통한 정보 노출**

[[ErrorMessageExposureCheck]]

**규칙: 사용자에게 표시되는 오류 메시지에는 시스템 내부 경로, 버전 정보, 스택 추적(Stack Trace) 등 민감 정보를 포함하지 않아야 한다.**

#### ❌ 잘못된 예


```c
void connect_db() {
    if (db_handle == NULL) {
        // ❌ 실제 서버 경로와 DB 라이브러리 버전을 노출
        printf("Error: Cannot find DB at /usr/local/mysql/data/config.db. (v5.7.1)");
    }
}
```

#### ✅ 바른 예


```c
void connect_db() {
    if (db_handle == NULL) {
        // ✅ 사용자에게는 추상적인 에러 메시지만 전달하고, 상세 내용은 로그에 남김
        printf("Error: 서비스에 연결할 수 없습니다. 관리자에게 문의하세요.");
        log_error("DB Connection Fail: /usr/local/mysql/data/config.db");
    }
}
```

### **2) CWE-367: 검사 시점과 사용 시점의 경쟁 조건 (TOCTOU)**

[[TOCTOUCheck]]

**규칙: 자원(파일 등)에 접근하기 전 검사한 상태가 사용 시점까지 유지된다고 가정하지 말고, 원자적(Atomic) 연산을 사용해야 한다.**

#### ❌ 잘못된 예


```c
void process_file(char *path) {
    if (access(path, W_OK) == 0) { // ❌ 검사 시점
        // 이 사이에 공격자가 파일을 심볼릭 링크로 교체 가능
        FILE *fp = fopen(path, "w"); // ❌ 사용 시점
        fputs("data", fp);
    }
}
```

#### ✅ 바른 예


```c
void process_file(char *path) {
    // ✅ 파일을 열 때 독점적 권한을 확인하거나, 열린 파일 핸들을 기반으로 작업
    FILE *fp = fopen(path, "wx"); // 'x'는 파일이 이미 존재하면 실패하도록 함
    if (fp != NULL) {
        fputs("data", fp);
        fclose(fp);
    }
}
```

### **3) CWE-369: 0으로 나누기 (Divide By Zero)**

[[DivideByZeroCheck]]

**규칙: 나눗셈 연산 시 제수(Divisor)가 0인지 반드시 확인해야 한다.**

#### ❌ 잘못된 예


```c
int calculate_average(int total, int count) {
    return total / count; // ❌ count가 0일 경우 런타임 오류 발생
}
```

#### ✅ 바른 예


```c
int calculate_average(int total, int count) {
    if (count == 0) { // ✅ 제수 0 체크
        return 0;
    }
    return total / count;
}
```

### **4) CWE-390: 오류 상황 감지 후 조치 부재**

[[ErrorHandlingActionCheck]]

**규칙: 함수 호출 후 에러 코드가 반환된 경우, 이를 무시하지 말고 적절한 복구 또는 예외 처리를 수행해야 한다.**

#### ❌ 잘못된 예


```c
void update_config() {
    int result = setuid(0); 
    if (result == -1) {
        // ❌ 에러를 감지만 하고 아무런 조치 없이 진행 (권한 획득 실패 상태)
    }
    execute_privileged_task();
}
```

#### ✅ 바른 예


```c
void update_config() {
    if (setuid(0) == -1) {
        // ✅ 에러 발생 시 즉시 실행을 중단하고 로그 기록
        perror("Permission Error");
        exit(EXIT_FAILURE);
    }
    execute_privileged_task();
}
```

### **5) CWE-489: 활성화된 디버그 코드**

[[ActiveDebugCodeCheck]]

**규칙: 배포(Production) 버전의 소스코드에는 디버깅을 위해 작성한 테스트 코드나 백도어가 포함되지 않아야 한다.**

#### ❌ 잘못된 예


```c
void login(char *id) {
    // ❌ 개발 편의를 위해 남겨둔 디버그용 관리자 로그인
    if (strcmp(id, "DEBUG_ADMIN") == 0) {
        grant_access();
        return;
    }
    process_normal_login();
}
```

#### ✅ 바른 예


```c
void login(char *id) {
#ifdef DEBUG_MODE
    if (strcmp(id, "DEBUG_ADMIN") == 0) {
        grant_access();
        return;
    }
#endif
    process_normal_login(); // ✅ 실제 배포 시에는 DEBUG_MODE를 끄고 빌드
}
```

### **6) CWE-495: Public 메소드에서 Private 데이터 반환**
[공용]
[[PureVirtualInitCheck]]

**규칙: 클래스나 모듈 내부의 private 배열/구조체에 대한 참조를 직접 반환하지 말고, 복사본을 반환해야 한다.**

#### ❌ 잘못된 예


```c
class UserData {
private:
    char internal_key[10];
public:
    char* getKey() {
        return internal_key; // ❌ 내부 주소를 직접 반환하여 외부에서 수정 가능하게 함
    }
};
```

#### ✅ 바른 예


```c
class UserData {
private:
    char internal_key[10];
public:
    void getKey(char *out_buf) {
        // ✅ 값을 복사하여 전달함으로써 내부 구조 보호
        memcpy(out_buf, internal_key, 10);
    }
};
```

### **7) CWE-496: Public 데이터를 Private 배열에 할당**
[공용]
[[PublicDataAssignedCheck]]

**규칙: 외부에서 전달받은 배열이나 객체의 참조를 내부 private 필드에 직접 할당하지 말고, 복사하여 저장해야 한다.**

#### ❌ 잘못된 예


```c
class MySettings {
private:
    int *config_values;
public:
    void setValues(int *input) {
        config_values = input; // ❌ 외부 포인터를 직접 할당 (외부에서 값 변경 시 내부 영향)
    }
};
```

#### ✅ 바른 예


```c
class MySettings {
private:
    int config_values[5];
public:
    void setValues(int *input) {
        // ✅ 값을 복사하여 저장 (독립성 유지)
        for(int i=0; i<5; i++) config_values[i] = input[i];
    }
};
```

### **8) CWE-497: 시스템 데이터 정보 노출**

[[SystemInformationExposureCheck]]

**규칙: 시스템 환경 변수, 설정 파일 경로, 프로세스 ID 등의 시스템 데이터가 권한이 없는 사용자에게 노출되지 않도록 해야 한다.**

#### ❌ 잘못된 예


```c
void show_status() {
    // ❌ 전체 환경 변수를 사용자 화면에 출력
    extern char **environ;
    for (char **env = environ; *env; ++env) printf("%s\n", *env);
}
```

#### ✅ 바른 예


```c
void show_status() {
    // ✅ 필요한 비민감 정보(예: 업타임)만 선택적으로 제공
    printf("Server Status: Running\n");
    printf("Version: 1.0.2\n");
}
```

// ✅ 로직에 대한 설명만 남김

if (login(id, pw)) { ... }

### **10) CWE-628: 잘못된 인자로 함수 호출**

[[IncorrectArgumentCheck]]

**규칙: 함수 호출 시 인자의 타입, 순서, 개수가 함수의 선언 및 정의와 일치해야 한다.**

#### ❌ 잘못된 예


```c
// 정의: void set_permission(int user_id, int level);
set_permission(1, 100); // 정상 호출
set_permission(100, 1); // ❌ 인자 순서가 바뀌어 잘못된 권한 부여
```

#### ✅ 바른 예


```c
// ✅ 인자의 의미를 명확히 알 수 있도록 변수 사용 및 검증
int admin_id = 1;
int admin_level = 100;
set_permission(admin_id, admin_level);
```

### **11) CWE-676: 위험한 함수 사용**
[[공통]]
[[FixedAddressAssignmentCheck]]

**규칙: 버퍼 오버플로우에 취약한 함수(strcpy, gets, strcat 등) 대신 경계값을 확인하는 안전한 함수(strncpy, fgets, strncat 등)를 사용해야 한다.**

#### ❌ 잘못된 예


```c
void input_name() {
    char name[10];
    gets(name); // ❌ 버퍼 크기를 무시하고 입력받음 (Buffer Overflow)
}
```

#### ✅ 바른 예


```c
void input_name() {
    char name[10];
    fgets(name, sizeof(name), stdin); // ✅ 버퍼 크기 내에서만 입력받음
}
```

### **12) CWE-755: 예외 조건의 부적절한 처리**

[[ExceptionHandlingCheck]]

**규칙: 예외가 발생할 수 있는 코드 영역은 Try-Catch 등으로 감싸고, 예외 발생 시 자원 해제 및 상태 복구를 수행해야 한다.**

#### ❌ 잘못된 예


```c
// ❌ 소멸자에서 예외가 발생하여 프로그램이 비정상 종료됨
MyClass::~MyClass() {
    throw std::runtime_error("Error"); 
}
```

#### ✅ 바른 예


```c
// ✅ 예외를 내부에서 처리하여 전파되지 않도록 방어
MyClass::~MyClass() {
    try {
        cleanup();
    } catch (...) {
        // 로그 기록 후 예외가 밖으로 나가지 않게 차단
    }
}
```
