
## 1. 제품 개요 및 분석 워크플로우

ArqaStatic은 WPF(UI) 기반의 대시보드를 통해 직관적인 분석 흐름을 제공하는 C/C++ 전문 정적 분석 도구입니다.

- **프로젝트 설정 및 파일 등록:** 메인 화면에서 프로젝트를 생성한 후, 드래그 앤 드롭이나 '파일 추가' 버튼을 통해 분석할 C/C++ 소스 파일들을 목록에 등록합니다.

- **분석 환경 구성:** '체커 및 옵션', '포함 경로(Include)', '전처리 매크로(Macro)' 메뉴를 지원하여 프로젝트별 맞춤형 환경 설정을 수행합니다.

- **분석 실행 및 탐지:** '분석 실행 ▶' 버튼으로 탐색을 시작하면 도구가 코드를 스캔합니다. 완료 후 데이터그리드(표) 형태로 파일명, 위반 룰, 코드 라인, 오류 메시지 등이 명확하게 출력됩니다.

- **상세 분석 및 리포트 내보내기:** 탐지된 항목을 더블클릭하거나 '상세 보기'를 누르면 문제 코드의 컨텍스트를 확인할 수 있습니다. 최종 결과는 PDF, Excel, HTML 형태의 보고서로 즉시 내보낼 수 있습니다.
    

## 2. 4대 핵심 분석 엔진 및 원천 기술

코드를 단순 텍스트로 매칭하는 한계를 넘어, 어휘 수준부터 심층 실행 경로까지 총 4가지 계층으로 나누어 입체적인 분석을 수행합니다.

### 4가지 계층별 분석 방식

|**분석 계층**|**설명**|**탐지 영역**|
|---|---|---|
|**Lexical** (어휘 분석)|코드의 의미를 해석하기 전 단계의 분석|텍스트 및 토큰 수준에서 발생하는 원초적인 문제|
|**AST** (구문 트리 분석)|소스코드를 구조적인 트리 형태로 변환하여 분석|잘못된 코딩 스타일, 문법적 오류 검출|
|**CFG** (제어 흐름 분석)|프로그램이 실행되는 분기 구조를 따라가며 분석|If, Switch, 루프(Loop) 등에서 발생 가능한 흐름 오류|
|**Path-Sensitive** (경로 민감 분석)|메모리 할당부터 해제까지 특정 실행 경로를 추적|변수의 상태 및 값의 변화를 감시하여 치명적인 런타임/보안 취약점 탐지|

### 오탐(False Positive) 최소화를 위한 핵심 기술

> **CTU (Cross-Translation Unit) 인터프로시저럴 분석 엔진**
> 
> 단일 파일 내부 검사에 그치지 않고, 여러 파일(TU)에 걸쳐 있는 함수 호출과 변수 참조 관계를 유기적으로 추적합니다. 분석 시 .ast 파일과 externalDefMap.txt(Index) 매핑 파일을 사전에 생성한 뒤 파일 간 연관 관계를 병렬로 처리하여 정밀도를 대폭 끌어올립니다.

> **Z3 솔버(Solver) 엔진**
> 
> 복잡한 조건문식이나 수학적 논리식을 분석할 때, 실제 구동 시 발생 불가능한 경로(Dead Path)를 수학적으로 증명해 냅니다. 이 엔진을 통해 제어 흐름상 유효하지 않은 결함을 걸러내어 오탐율을 획기적으로 줄입니다.

## 3. 진단 규칙 및 보안 규격 (Rule Set)

산업 표준 보안 규격 및 무기체계 개발 표준을 완벽히 맵핑하여 진단합니다.

- **CWE (Common Weakness Enumeration) 취약점:** 소프트웨어 보안을 위협하는 치명적인 결함을 탐지합니다.
    
    - _주요 탐지 항목:_ CWE-22(경로 조작), CWE-78(OS 명령어 삽입), CWE-89(SQL 인젝션), CWE-119(버퍼 오버플로우), CWE-476(NULL 포인터 역참조), CWE-415/416(이중 해제 및 Use-After-Free) 등
        
- **DAPA (방위사업청) 코딩 규칙:** 무기체계 소프트웨어 개발 표준에 부합하는 공통 규칙 1~7항(스타일, 초기화, 조건식, 포인터 등) 및 C/C++ 전용 코딩 규칙을 완벽히 지원합니다.
    

## 4. AI 기반 자동화 및 소프트웨어 메트릭 분석

### RAG 기법 기반 AI 코드 수정 제안

단순히 결함 위치를 알려주는 것에 그치지 않고, 생산성 향상과 비용 절감을 위해 AI 원인 분석 기능을 제공합니다.

1. **동적 가이드라인 발췌:** 룰 위반 시, 도구 내부에 내장된 보안 규격 마크다운(.md) 문서에서 해당 규칙의 가이드라인을 RAG(검색 증강 생성) 방식으로 동적 탐색합니다.
    
2. **컨텍스트 전달 및 분석:** 탐지된 소스코드 스니펫과 추출된 규격 문서를 조합하여 AI 솔루션에 전달합니다.
    
3. **구조화된 보고서 출력:** AI는 분석 결과 구조를 JSON 형태로 반환하며, UI 대시보드 상에 마크다운 리포트 형태로 시각화합니다.
    
    - _제공 데이터:_ 문제 요약(Summary), 근본 원인(Cause), 수정 제안(Suggestion), 즉시 적용 가능한 수정 코드(Fixed Code)
        
4. **보안성 및 비용 절감:** 사내망 혹은 로컬 VLLM 서버(오픈소스 모델 gpt-oss-20b 등)와 연동하도록 설계되어, 소스코드의 외부 유출을 원천 차단하고 값비싼 상용 API 호출 비용을 절감합니다.
    

### 구조적 코드 메트릭 및 결합도(Coupling) 분석

Final_CallGraph.json을 기반으로 프로그램의 복잡성과 유지보수성을 정량화하여 가장 위험도가 높은 함수(Top Risk Function)를 도출합니다.

- **측정 지표:** 순환 복잡도(Cyclomatic Complexity), 비주석 라인 수(Ncloc), 함수 결합도(Fan-In / Fan-Out), 호출 깊이(Nesting) 등
    
- **시각화:** 정량화된 수치적 지표를 시각적인 대시보드와 PDF 리포트로 제공하여 구조적 품질 관리를 돕습니다.
    

## 5. 기업 및 관공서 맞춤형 특화 기능

- **HWPX 오탐 보고서 1초 자동 생성:** 탐지된 결함 중 개발자가 오탐(False Alarm)으로 판정한 항목에 대해 소명을 자동화합니다. 개발자가 UI에서 오탐 사유를 입력하면 해당 코드 영역을 자동으로 이미지(PNG) 캡처하고, 내장된 한글 문서(HWPX) 템플릿 XML 내부의 변수(CWE_ID, MESSAGE 등)를 직접 치환하여 관공서 제출용 증빙 문서를 즉시 생성합니다.
    
- **비교(Diff) 리포트:** 과거 분석 이력과 현재 상태를 대조하여 신규 발생(New), 해결됨(Fixed), 변동 없음(Unchanged) 항목으로 분류합니다. 코드 변경 사항은 전용 Diff 뷰어 창에서 초록색과 빨간색 하이라이팅으로 직관적으로 비교 분석할 수 있습니다.
