좋습니다. 이제 컴파일러의 '꽃'이자 '두뇌'라고 불리는 **최적화 (Optimization)** 단계입니다.

중간 코드 생성(ICG)을 통해 우리는 언어와 기계로부터 독립된 **중간 표현(IR)**과 **제어 흐름 그래프(CFG)**를 얻었습니다. 이 IR은 '올바른' 코드이긴 하지만, '비효율적인' 코드일 수 있습니다.

최적화 단계의 목표는 프로그램의 '의미'는 절대 바꾸지 않으면서, 프로그램을 **더 빠르거나(시간)**, **더 작게(공간)** 만드는 것입니다.

### 🎯 최적화의 목표

- **입력 (Input):** 비효율적일 수 있는 IR (그리고 이를 표현한 CFG)
    
- **출력 (Output):** 더 효율적인(빠르고 작은) IR (그리고 수정된 CFG)
    
- **핵심 원칙:** **의미 보존 (Semantics Preservation)**. 최적화 전과 후의 프로그램 실행 결과는 _반드시_ 같아야 합니다.
    

---

### 1. 🔬 핵심 이론: 데이터 흐름 분석 (Data-Flow Analysis)

최적화는 '감'으로 하는 것이 아닙니다. **제어 흐름 그래프(CFG)**를 바탕으로 데이터가 프로그램 내에서 어떻게 흐르고 사용되는지 **수학적으로 분석**하는 과정을 거칩니다. 이를 **데이터 흐름 분석**이라고 합니다.

CFG의 각 지점(기본 블록의 시작과 끝)에서 다음과 같은 질문에 답합니다.

- **활성 변수 분석 (Liveness Analysis):** "이 지점에서 변수 `x`의 값이 나중에 _또_ 사용되는가?"
    
    - → 만약 `x`가 '죽었다면(dead)' (다시는 사용되지 않는다면), `x`에 값을 저장하는 코드는 '죽은 코드'이므로 제거할 수 있습니다.
        
- **도달 정의 분석 (Reaching Definitions):** "이 지점에서 변수 `x`의 값은 어떤 할당문(예: `x=10` 또는 `x=y`)으로부터 온 것인가?"
    
    - → `x`의 값이 항상 `10`임이 보장된다면, `x`를 `10`으로 바꿀 수 있습니다. (상수 전파)
        

---

### 2. ✨ 대표적인 최적화 기법 (예시)

이러한 데이터 흐름 분석을 바탕으로 수십, 수백 가지의 최적화 기법이 적용됩니다. 몇 가지 고전적이고 강력한 예를 들어보겠습니다.

#### A. 지역 최적화 (Local Optimization) - (하나의 기본 블록 내)

- **상수 폴딩 (Constant Folding) / 상수 전파 (Constant Propagation)**
    
    - 컴파일 시점에 계산할 수 있는 것을 미리 계산합니다.
        
    - **Before:**
        
        ```
        x = 10;
        y = 20;
        z = x + y;
        ```
        
    - **After (전파 & 폴딩):**
        
        ```
        x = 10;
        y = 20;
        z = 30;  // (x+y)가 30으로 '폴딩'됨
        ```
        
- **대수적 간소화 (Algebraic Simplification)**
    
    - 수학 공식을 이용해 코드를 간소화합니다.
        
    - **Before:** `x = y + 0;` (0 더하기)
        
    - **After:** `x = y;`
        
    - **Before:** `a = b * 2;`
        
    - **After:** `a = b << 1;` (곱셈을 더 빠른 쉬프트 연산으로)
        

---

#### B. 전역 최적화 (Global Optimization) - (함수 전체 CFG 대상)

- **죽은 코드 제거 (Dead Code Elimination)**
    
    - '활성 변수 분석' 등을 통해 아무런 영향을 주지 않는 코드를 제거합니다.
        
    - **Before:**
        
        ```
        x = 10;
        y = 5;       // 'y'는 어디에서도 사용되지 않음 (dead)
        return x * 2;
        ```
        
    - **After:**
        
        ```
        x = 10;
        return x * 2;
        ```
        
- **공통 부표현식 제거 (Common Subexpression Elimination, CSE)**
    
    - 여러 번 계산되는 동일한 식을 한 번만 계산하도록 임시 변수에 저장합니다.
        
    - **Before:**
        
        ```
        a = (b * c) + d;
        x = (b * c) - e;
        ```
        
    - **After:**
        
        ```
        t1 = b * c;  // 공통 표현식 'b*c'
        a = t1 + d;
        x = t1 - e;
        ```
        
- **루프 불변 코드 이동 (Loop-Invariant Code Motion)**
    
    - **루프 최적화의 꽃입니다.** 반복문 안에서 결과가 변하지 않는 코드를 반복문 _밖_으로 빼냅니다.
        
    - **Before:**
        
        ```
        for (i = 0; i < 100; i++) {
            x = y + z;  // i와 관계없이 항상 같은 값!
            a[i] = x;
        }
        ```
        
    - **After:**
        
        ```
        t1 = y + z;  // 밖으로 이동! (100번 -> 1번)
        for (i = 0; i < 100; i++) {
            a[i] = t1;
        }
        ```

### 3. ⚖️ 최적화의 트레이드오프

최적화는 공짜가 아닙니다. 최적화를 많이 할수록(예: 컴파일 옵션 `-O3`) 컴파일 시간은 오래 걸리지만, 실행 파일의 성능은 좋아집니다. 반대로 최적화를 끄면(`-O0`, 디버깅 시) 컴파일은 매우 빠르지만 성능은 나빠집니다.

### 요약 및 다음 단계

최적화 단계를 거쳐, 우리의 IR은 '올바를' 뿐만 아니라 '매우 효율적인' 형태로 다듬어졌습니다.

이제 모든 준비가 끝났습니다. 이 효율적인 **최종 IR**을 가지고, 드디어 실제 목표 기계(예: Intel x86, ARM)가 알아들을 수 있는 언어로 번역하는 마지막 단계인 **코드 생성 (Code Generation)**으로 넘어갑니다.
