### [조건식-가] 부동소수점 동등성 비교 금지
- checkerName: NoFloatEqualityCheck
- category: Conditional Expression
- severity: MAJOR
- summary: "부동소수점(float, double) 타입은 '==' 연산자로 직접적인 동등성 비교를 수행해서는 안 됩니다."
- violation: `if (float_a == float_b)`
- correction: `if (fabs(float_a - float_b) < EPSILON)`

### [조건식-나] 항상 참/거짓인 조건문 금지
- checkerName: AlwaysConstantConditionCheck
- category: Conditional Expression
- severity: WARNING
- summary: "결과가 항상 참(true) 또는 거짓(false)으로 정해져 있는 불필요한 조건문을 사용하지 않습니다."
- violation: `unsigned int x; if (x < 0) { ... }`
- correction: "불필요한 조건문 제거"

### [조건식-다] 도달할 수 없는 switch case 레이블 금지
- checkerName: UnreachableSwitchCaseCheck
- category: Conditional Expression
- severity: WARNING
- summary: "switch문의 변수 타입으로 인해 절대 만족될 수 없는 case 레이블은 사용하지 않습니다."
- violation: `unsigned int n; switch(n) { case -1: ... }`
- correction: "도달할 수 없는 case 레이블 제거"

### [조건식-라] switch문에 논리 연산 사용 금지
- checkerName: SwitchStyleCheck
- category: Conditional Expression
- severity: WARNING
- summary: "결과가 0 또는 1인 논리 연산을 switch문의 조건식으로 사용하는 것은 if-else 구조에 비해 가독성이 떨어집니다."
- violation: `switch (option == pick) { ... }`
- correction: `if (option == pick) { ... } else { ... }`

### [조건식-마] 도달할 수 없는 코드 금지 (Dead Code)
- checkerName: UnreachableCodeCheck, clang-analyzer-core.uninitialized.DeadCode
- category: Control Flow
- severity: MAJOR
- summary: "break, return, throw 키워드 뒤와 같이 절대 실행될 수 없는 코드를 작성하지 않습니다."
- violation: `break; n = n + 1;`
- correction: "실행될 수 없는 `n = n + 1;` 구문 제거"
