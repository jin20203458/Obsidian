

### [초기화-가] char 배열 크기
- checkerName: CharArrayStringSizeCheck
- category: Initialization
- severity: ERROR
- summary: "문자열 리터럴로 char 배열 초기화 시, 배열 크기는 문자열 길이에 널 문자(\\0)를 더한 값 이상이어야 합니다."
- violation: `char h[5] = "Hello";` (6바이트 필요)
- correction: `char h[6] = "Hello";`

### [초기화-나] 변수 사용 전 초기화
- checkerName: clang-analyzer-core.uninitialized.Assign, InitBeforeUseCheck
- category: Initialization
- severity: ERROR
- summary: "모든 지역 변수는 값을 읽기 전에 반드시 초기화되어야 합니다."
- violation: `int i; j = i + 1;`
- correction: `int i = 0; j = i + 1;`

### [초기화-다] 초기화되지 않은 변수 주소를 const 포인터 인자로 전달 금지
- checkerName: clang-analyzer-core.CallAndMessage, UninitializedAddressToConstParamCheck
- category: Initialization
- severity: MAJOR
- summary: "초기화되지 않은 변수의 주소를 const 포인터 매개변수를 받는 함수에 전달할 수 없습니다."
- violation: `int y; foo(&y);` (foo가 const int*를 받을 때)
- correction: `int y = 0; foo(&y);`
