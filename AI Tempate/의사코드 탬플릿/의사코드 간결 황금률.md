 

## 최종 황금률: Obsidian/KaTeX 의사코드 작성을 위한 절대 규칙 (개정판)

## 제0규칙: 핵심 알고리즘만 간결하게 표현

의사코드는 **가장 중요한 로직의 흐름**만을 명확하게 보여주어야 합니다. 생성자, 옵션 설정, 단순한 getter/setter, 간단한 유틸리티 함수 등은 의사코드에 포함하지 않습니다.

- **포함하지 않아야 할 것**:
    
    - 생성자에서 옵션을 저장하는 로직 (`Constructor`, `storeOptions`)
        
    - 단순 반환 함수 (`IsAsciiOnly`, `HasWhitespace` 같은 1-2줄 체크 함수)
        
    - 전처리기 콜백 등록과 같은 프레임워크 연동 코드 (`RegisterPPCallbacks`)
        
    - 지나치게 자세한 주변 로직
        
- **포함해야 할 것**:
    
    - 알고리즘의 핵심 제어 흐름 (`HandleInclusionDirective`)
        
    - 중요한 검증 로직 (복잡한 조건문이 있는 함수)
        
    - 비즈니스 로직의 핵심 (경로 검증, 문자 검사 등)
        
- **목표**: 3-5개의 `\Procedure` 블록으로 전체 알고리즘을 표현합니다.
    

## 제1규칙: 허용된 기본 명령어 외에는 일절 사용 금지

`algorithmic` 환경 내에서는 아래에 명시된 극소수의 기본 명령어만 사용해야 합니다. 목록에 없는 명령어(`\Call`, `\Comment`, `\Statex`, `\ElsIf` 등)는 KaTeX 파서가 해석하지 못하므로 절대 사용해선 안 됩니다.

- **허용 명령어 목록**:
    
    - `\begin{algorithm}` / `\end{algorithm}`
        
    - `\caption{...}`
        
    - `\begin{algorithmic}` / `\end{algorithmic}`
        
    - `\Procedure{Name}{Args}` / `\EndProcedure`
        
    - `\State ...`
        
    - `\If{Condition}` / `\EndIf`
        
    - `\For{Loop Condition}` / `\EndFor`
        
    - `\While{Condition}` / `\EndWhile`
        
    - `\Return ...`
        
- **금지 명령어**: `\Call`, `\Comment`, `\Statex`, `\ElsIf`, `\Else`, `\Function`, 기타 확장 명령
    
- **`\ElsIf` 및 `\Else` 사용 금지**: KaTeX 환경에서 `\Else`와 `\ElsIf`는 지원되지 않거나 불안정합니다. 대신 여러 개의 독립적인 `\If` 블록을 사용하거나, 설명이 필요하면 중첩된 `\If`를 사용합니다.
    

## 제2규칙: `\begin{algorithmic}`에는 어떠한 옵션도 금지

줄 번호를 붙이는 등의 옵션(`[1]`)은 KaTeX에서 지원하지 않으며 즉시 오류를 발생시킵니다. 항상 옵션 없이 사용해야 합니다.

- **나쁜 예**: `\begin{algorithmic}[1]`
    
- **올바른 예**: `\begin{algorithmic}`
    

## 제3규칙: 제어문 조건부는 '순수 텍스트' 또는 '안전한 수학 모드'로만 작성

`\If`, `\For`, `\While`의 조건부 `{...}`는 오류의 가장 큰 원인이므로, 아래 원칙을 반드시 지켜야 합니다.

1. **원칙 1 (가장 안전):** 조건은 기호 없이 완전한 영어 문장으로 서술합니다.
    
2. **원칙 2 (제한적 허용):** 수학적/논리적 비교가 필요할 경우, 표현식 전체를 `$...$`로 감싸고, 그 안에는 **알파벳, 숫자, 그리고 `< > = + - * /` 같은 기본 연산자만** 사용합니다.
    

- **절대 금지 기호 (제어문 조건부에서)**:
    
    - 백슬래시 (`\`)
        
    - 백틱 (`` ` ``)
        
    - 밑줄 (`_`)
        
    - 달러 기호 (`$`)
        
    - 해시 (`#`)
        
    - 따옴표 (`"`, `'`)
        
    - 대괄호/중괄호/소괄호를 제외한 모든 프로그래밍 특수 문자
        
- **나쁜 예**:
    
    - `\If{$Raw.Inner$ contains` `}`
        
    - `\If{$c$ is one of` _`,` .`,` /`}`
        
    - `\If{path ends with "/"}`
        
    - `\If{character is space tab newline or other whitespace}` (너무 장황)
        
- **올바른 예**:
    
    - `\If{the path contains a backslash}`
        
    - `\If{the character is a special symbol}`
        
    - `\If{$A[i] < \text{THRESHOLD}$}`
        
    - `\If{the path ends with a slash}`
        
    - `\If{character is whitespace}` (간결하게)
        

## 제4규칙: `\text{...}` 명령어는 `\State` 뒤에서만, 그리고 제한적으로 사용

`\text{...}`는 제어문 조건부에서 파싱 오류를 유발합니다. 오직 `\State` 명령어 뒤에서 코드의 가독성을 높이기 위한 목적으로만 사용해야 합니다.

- **나쁜 예**: `\If{\text{Is Invalid}}`
    
- **올바른 예**: `\State $variable \gets \text{new Object()}$`
    
- **올바른 예**: `\If{the object is invalid}` (조건부는 순수 텍스트로)
    

## 제5규칙: 주석은 어떤 형태로든 절대 금지

코드 블록 내부에 주석을 남기는 모든 시도는 렌더링 오류의 원인이 됩니다. 의사코드는 주석 없이, 로직의 흐름만으로 이해할 수 있도록 명료하게 작성해야 합니다.

## 제6규칙: 간결성 유지 - 불필요한 중복과 장황한 설명 금지

의사코드는 알고리즘의 본질을 드러내는 것이 목적입니다.

- 각 `\Procedure`는 핵심 로직만 포함합니다.
    
- 반복되는 패턴(`Report error for...`)은 최소화하고, 필요하면 `\State $\text{reportError}(...)$` 한 줄로 통합합니다.
    
- 단순 체크 로직은 별도의 프로시저로 분리하지 않고, 메인 로직 안에서 `\If` 조건으로 간결하게 표현합니다.

## 검증된 최종 템플릿 (개정판)

```
\begin{algorithm}
\caption{Algorithm Title}
\begin{algorithmic}
  \Procedure{MainFunction}{$param1, param2$}
    \State $result \gets \text{initialize()}$
    
    \If{$param1 > threshold$}
      \State $\text{processData}(param1)$
    \EndIf
    
    \For{$i \gets 1$ \To $n$}
      \State $array[i] \gets array[i] + 1$
    \EndFor
    
    \State \Return $result$
  \EndProcedure

  \Procedure{HelperFunction}{$data$}
    \While{$data$ is not empty}
      \State $element \gets \text{extract}(data)$
      \State $\text{process}(element)$
    \EndWhile
  \EndProcedure
\end{algorithmic}
\end{algorithm}
```
