알겠습니다. 여러 번의 시행착오를 통해 얻은 교훈을 바탕으로, 앞으로 그 어떤 AI 모델이라도 혼동 없이 완벽하게 이해하고 따를 수 있도록, 예외 없는 **최종 황금률(The Final Golden Rules)**을 새롭게 정립하겠습니다.

이전의 가이드라인과 실제 발생했던 오류(`\`, `` ` `` 문자 문제 등)를 모두 종합하여, Obsidian/KaTeX 환경에서 의사코드 렌더링 오류를 원천적으로 차단하기 위한 가장 엄격하고 명확한 지침입니다.

## 최종 황금률: Obsidian/KaTeX 의사코드 작성을 위한 절대 규칙

## 제1규칙: 허용된 기본 명령어 외에는 일절 사용 금지

`algorithmic` 환경 내에서는 아래에 명시된 극소수의 기본 명령어만 사용해야 합니다. 목록에 없는 명령어(`\Call`, `\Comment`, `\Statex`, `\ElsIf` 등)는 KaTeX 파서가 해석하지 못하므로 절대 사용해선 안 됩니다.

- **허용 명령어 목록**:
    
    - `\begin{algorithm}` / `\end{algorithm}`
        
    - `\caption{...}`
        
    - `\begin{algorithmic}` / `\end{algorithmic}`
        
    - `\Procedure{Name}{Args}` / `\EndProcedure`
        
    - `\State ...`
        
    - `\If{Condition}` / `\EndIf`
        
    - `\For{Loop Condition}` / `\EndFor`
        
    - `\While{Condition}` / `\EndWhile`
        
    - `\Return ...`
        
- **나쁜 예**:
    
    - `\Call{MyFunction}{args}`
        
    - `\Comment{This is a comment}`
        
    - `\If{...} \State ... \ElsIf{...}`
        
- **올바른 예**:
    
    - `\State $MyFunction(args)$` (함수 호출은 `\State`와 일반 텍스트로 표현)
        
    - 주석은 사용하지 않음
        
    - `\If{...} \State ... \EndIf` (첫 번째 조건)  
        `\If{...} \State ... \EndIf` (`else if` 대신 별도의 `If`문 사용)
        

## 제2규칙: `\begin{algorithmic}`에는 어떠한 옵션도 금지

줄 번호를 붙이는 등의 옵션(`[1]`)은 KaTeX에서 지원하지 않으며 즉시 오류를 발생시킵니다. 항상 옵션 없이 사용해야 합니다.

- **나쁜 예**: `\begin{algorithmic}[1]`
    
- **올바른 예**: `\begin{algorithmic}`
    

## 제3규칙: 제어문 조건부는 '순수 텍스트' 또는 '안전한 수학 모드'로만 작성

`\If`, `\For`, `\While`의 조건부 `{...}`는 오류의 가장 큰 원인이므로, 아래 원칙을 반드시 지켜야 합니다.

1. **원칙 1 (가장 안전):** 조건은 기호 없이 완전한 영어 문장으로 서술합니다.
    
2. **원칙 2 (제한적 허용):** 수학적/논리적 비교가 필요할 경우, 표현식 전체를 `$...$`로 감싸고, 그 안에는 **알파벳, 숫자, 그리고 `< > =` 와 같은 기본 비교 연산자만** 사용합니다.
    

- **절대 금지 기호 (수학 모드 외부)**:
    
    - 백슬래시 (`\`)
        
    - 백틱 (`` ` ``)
        
    - 밑줄 (`_`), 달러 기호 (`$`), 해시 (`#`) 등 LaTeX 특수 문자
        
    - 프로그래밍에 사용되는 대부분의 특수 구두점
        
- **나쁜 예**:
    
    - `\If{$Raw.Inner$ contains` `}` (백슬래시, 백틱 사용 오류)
        
    - `\If{$c$ is one of` _`,` .`,` /`}` (백틱 사용 오류)
        
    - `\If{path ends with "/"}` (따옴표 사용 오류)
        
- **올바른 예**:
    
    - `\If{the path contains a backslash}` (완전한 문장으로 서술)
        
    - `\If{the character is a special symbol}` (완전한 문장으로 서술)
        
    - `\If{$A[i] < \text{THRESHOLD}$}` (`$...$` 내부에 안전한 기호만 사용)
        
    - `\If{the path ends with a slash}` (따옴표를 제거하고 문장으로 서술)
        

## 제4규칙: `\text{...}` 명령어는 `\State` 뒤에서만, 그리고 제한적으로 사용

`\text{...}`는 제어문 조건부에서 파싱 오류를 유발합니다. 오직 `\State` 명령어 뒤에서 코드의 가독성을 높이기 위한 목적으로만 사용해야 합니다.

- **나쁜 예**: `\If{\text{Is Invalid}}`
    
- **올바른 예**: `\State $variable \gets \text{new Object()}$`
    
- **올바른 예**: `\If{the object is invalid}` (조건부는 순수 텍스트로)
    

## 제5규칙: 주석은 어떤 형태로든 절대 금지

코드 블록 내부에 주석을 남기는 모든 시도는 렌더링 오류의 원인이 됩니다. 의사코드는 주석 없이, 로직의 흐름만으로 이해할 수 있도록 명료하게 작성해야 합니다.
