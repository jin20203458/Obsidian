네, 정보처리기사 시험에 자주 출제되는 주요 공격 기법들을 목적과 특징에 따라 표로 정리해 드리겠습니다. 키워드 중심으로 암기하시면 문제 풀이에 도움이 될 것입니다.

## 1. 서비스 거부(DoS) 및 분산 서비스 거부(DDoS) 공격

시스템의 자원을 고갈시켜 정상적인 서비스를 방해하는 공격입니다.[shareknowledge-top.tistory](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​

|공격 기법|핵심 키워드 및 설명|
|---|---|
|**SYN Flooding**|TCP 연결 과정(3-way-handshake)의 취약점을 이용. SYN 패킷만 계속 보내고 ACK를 받지 않아 서버의 연결 대기 큐(Backlog Queue)를 가득 채움[shareknowledge-top.tistory](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​.|
|**UDP Flooding**|대량의 UDP 패킷을 임의의 포트로 전송하여 서버의 자원을 고갈시키는 공격[shareknowledge-top.tistory](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​.|
|**Ping of Death**|허용된 크기보다 훨씬 큰 ICMP 패킷을 잘게 분할하여 전송. 공격 대상은 분할된 패킷을 재조합하는 과정에서 과부하가 발생[sugoring-it.tistory+1](https://sugoring-it.tistory.com/141)​.|
|**Smurf Attack**|출발지 IP를 공격 대상의 IP로 위조하여 네트워크 전체에 ICMP Echo 요청을 브로드캐스팅. 수많은 ICMP 응답이 공격 대상에게 집중되도록 함[shareknowledge-top.tistory+1](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​.|
|**LAND Attack**|출발지 IP와 목적지 IP를 모두 공격 대상의 IP로 설정한 패킷을 전송하여 시스템이 자기 자신에게 계속 응답하게 만들어 부하를 유발[shareknowledge-top.tistory](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​.|
|**Teardrop Attack**|IP 패킷의 분할(Fragment) 순서를 의도적으로 뒤섞거나 중첩시켜 전송. 공격 대상이 패킷을 재조합하는 과정에서 오류가 발생하여 시스템이 다운됨[shareknowledge-top.tistory](https://shareknowledge-top.tistory.com/entry/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-5-%EB%B3%B4%EC%95%88-%EA%B3%B5%EA%B2%A9-%EA%B8%B0%EB%B2%95-%EC%B4%9D-%EC%A0%95%EB%A6%AC-4%EA%B3%BC%EB%AA%A9-%EC%A0%95%EB%B3%B4%EB%B3%B4%EC%95%88)​.|

## 2. 웹 기반 공격

웹 애플리케이션의 취약점을 이용하는 공격입니다.

| 공격 기법                                 | 핵심 키워드 및 설명                                                                                                                          |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **SQL Injection**                     | 악의적인 SQL 구문을 입력값으로 삽입하여 데이터베이스를 비정상적으로 조작(조회, 수정, 삭제)하는 공격.                                                                          |
| **XSS (Cross-Site Scripting)**        | 공격자가 악성 스크립트를 웹 페이지에 삽입. 다른 사용자가 해당 페이지를 열람하면 스크립트가 실행되어 쿠키나 세션 정보 등을 탈취[sugoring-it.tistory](https://sugoring-it.tistory.com/141)​. |
| **CSRF (Cross-Site Request Forgery)** | 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위(수정, 등록 등)를 특정 웹사이트에 요청하게 만드는 공격. 사용자의 인증된 세션을 악용.                                                      |

## 3. 정보 수집 및 권한 획득 공격

시스템 내부 정보를 탈취하거나 접근 권한을 획득하는 것을 목적으로 합니다.

| 공격 기법                                               | 핵심 키워드 및 설명                                                                                                                                                                                         |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Key Logger Attack**                               | 사용자의 키보드 입력을 탐지하여 ID, 패스워드 등 개인 정보를 가로채는 공격[velog+1](https://velog.io/@jjewwi12/%EC%A0%95%EB%B3%B4%EC%B2%98%EB%A6%AC%EA%B8%B0%EC%82%AC-%ED%95%84%EA%B8%B0-%EC%A0%95%EB%A6%AC-5%EA%B3%BC%EB%AA%A9)​. |
| **Spyware**                                         | 사용자 동의 없이 설치되어 개인 정보(금융 정보, 웹사이트 방문 기록 등)를 수집하여 공격자에게 전송하는 악성 프로그램.                                                                                                                                 |
| **Brute-Force Attack**<br>(무차별 대입 공격)               | 가능한 모든 문자 조합을 무작위로 시도하여 비밀번호나 암호화 키를 찾는 공격                                                                                                                                                          |
| **Session Hijacking**<br>(세션 하이재킹)                  | 사용자의 세션 ID를 가로채 인증된 사용자처럼 시스템에 접근하는 공격                                                                                                                                                              |
| **Phishing / Spear Phishing**<br>(피싱 / 스피어 피싱)      | 신뢰 기관을 사칭해 이메일이나 메시지를 보내고, 가짜 웹사이트로 유도해 정보를 탈취<br>스피어 피싱은 특정 개인이나 조직을 목표로 함                                                                                                                         |
| **Watering Hole**<br>(워터링 홀 공격)                     | 대상이 자주 방문하는 웹사이트를 미리 감염시켜, 방문 시 악성코드를 설치하는 표적 공격                                                                                                                                                    |
| **APT (Advanced Persistent Threat)**<br>(지능형 지속 위협) | 특정 목표를 대상으로 장기간 다양한 공격 기법을 동원해 지속적으로 정보를 수집하고 공격                                                                                                                                                    |
