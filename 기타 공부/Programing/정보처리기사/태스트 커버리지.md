## 

|커버리지 종류|설명|예시|
|---|---|---|
|**구문 커버리지 (Statement Coverage)**|모든 **문장**이 최소 1번 이상 실행되었는지 확인|if문 안/밖 코드 실행 여부|
|**결정 커버리지 (Decision Coverage)**|모든 **분기(조건식의 True/False)**가 최소 1번 이상 실행되었는지 확인|if 조건이 True일 때, False일 때 모두 실행|
|**조건 커버리지 (Condition Coverage)**|조건식 안의 **개별 조건**이 True/False 모두 실행되었는지 확인|`(A && B)`에서 A, B 각각 True/False|
|**조건/결정 커버리지 (Condition/Decision Coverage)**|조건 커버리지 + 결정 커버리지를 결합|분기와 개별 조건 모두 확인|
|**다중 조건 커버리지 (Multiple Condition Coverage)**|조건식의 모든 **조합**을 실행|`(A && B)` → (T,T), (T,F), (F,T), (F,F)|
|**경로 커버리지 (Path Coverage)**|가능한 모든 실행 경로를 테스트|if-else, 반복문 포함 모든 경로|

##  핵심 요약

- **테스트 커버리지 = 테스트의 “충분성”을 수치로 표현**
    
- **높을수록 좋지만**, 100% 커버리지가 곧 **버그 없음**을 의미하지는 않음
    
- 실무에서는 **구문 + 결정 커버리지**를 기본으로, 중요 시스템은 **조건/결정, 다중 조건 커버리지**까지 고려
