# GoF 디자인 패턴 23가지 한눈에 정리

GoF(Gang of Four) 디자인 패턴은 객체지향 설계에서 자주 사용되는 반복적인 문제 해결 방법을 23가지로 정리한 것입니다. 세 가지 카테고리로 분류됩니다: **생성(Creational), 구조(Structural), 행위(Behavioral)**

---

## 1. 생성 패턴 (총 5개)

- **싱글톤 (Singleton)**: 오직 하나의 인스턴스만 생성하여 공유
    
- **팩토리 메서드 (Factory Method)**: 객체 생성을 서브클래스에서 결정
    
- **추상 팩토리 (Abstract Factory)**: 관련 객체들 집합을 생성하는 인터페이스 제공
    
- **빌더 (Builder)**: 객체 생성 과정을 분리(복합 객체 단계별 생성)
    
- **프로토타입 (Prototype)**: 기존 인스턴스를 복사해 객체 생성
    

---

## 2. 구조 패턴 (총 7개)

- **어댑터 (Adapter)**: 인터페이스를 호환성 있게 변환
    
- **브리지 (Bridge)**: 구현과 추상화 분리하여 독립적으로 확장
    
- **컴포지트 (Composite)**: 전체-부분 트리 구조, 단일 객체와 복합 객체를 동일하게 취급
    
- **데코레이터 (Decorator)**: 객체에 새로운 책임 추가(확장)
    
- **퍼사드 (Facade)**: 복잡한 하위 시스템을 단순화된 인터페이스로 제공
    
- **플라이웨이트 (Flyweight)**: 메모리 절약을 위해 공유객체 사용
    
- **프록시 (Proxy)**: 접근 제어나 추가 기능을 위해 대리 객체 사용
    

---

## 3. 행위 패턴 (총 11개)

- **옵저버 (Observer)**: 상태 변화 시 자동으로 관련 객체에 통보
    
- **전략 (Strategy)**: 행위를 캡슐화, 런타임 동적 선택
    
- **상태 (State)**: 객체 내부 상태에 따라 행위 변화
    
- **템플릿 메서드 (Template Method)**: 알고리즘 구조 정의, 하위 클래스에서 구체적 단계 수정
    
- **책임 연쇄 (Chain of Responsibility)**: 여러 객체가 요청을 처리, 체인 구조로 연결
    
- **커맨드 (Command)**: 요청을 객체로 캡슐화, 실행, 취소, 재실행 가능
    
- **인터프리터 (Interpreter)**: 언어 문법과 해석을 위한 클래스 구조
    
- **반복자 (Iterator)**: 집합 객체의 요소를 순회하는 객체 제공
    
- **중재자 (Mediator)**: 객체 사이의 복잡한 상호작용 제어 및 캡슐화
    
- **메멘토 (Memento)**: 객체 상태 저장 및 복원(undo 등)
    
- **방문자 (Visitor)**: 알고리즘을 객체 구조에서 분리, 새로운 연산 추가 용이
