
### 결합도의 종류 (강한 → 약한 순)

| 결합도                            | 설명                                                      | 예시                         |
| ------------------------------ | ------------------------------------------------------- | -------------------------- |
| **내용 결합도 (Content Coupling)**  | 한 모듈이 다른 모듈의 내부 변수나 기능을 직접 참조하거나 수정하는 경우                | 모듈 A가 모듈 B의 지역 변수에 직접 접근   |
| **공통 결합도 (Common Coupling)**   | 여러 모듈이 전역 변수(Global Variable)를 공유하여 사용하는 경우             | 전역 배열을 여러 모듈이 동시에 읽고 쓰는 경우 |
| **외부 결합도 (External Coupling)** | 모듈이 외부에서 선언된 데이터(파일, 디바이스, 인터페이스 등)를 공유하는 경우            | 공용 인터페이스, 외부 장치 제어         |
| **제어 결합도 (Control Coupling)**  | 어떤 모듈이 다른 모듈의 내부 논리 흐름을 제어하기 위해 제어 신호(플래그, 코드)를 전달하는 경우 | 함수 호출 시 플래그 값 전달하여 분기 제어   |
| **스탬프 결합도 (Stamp Coupling)**   | 모듈 간에 자료 구조(레코드, 구조체, 배열 등)를 통째로 전달하는 경우                | 구조체 전체를 매개변수로 전달           |
| **자료 결합도 (Data Coupling)**     | 모듈 간에 필요한 데이터만 매개변수로 전달하는 경우                            | 함수 호출 시 필요한 값만 인자로 전달      |

### 응집도의 종류 (낮은 → 높은 순)

|응집도|설명|예시|
|---|---|---|
|**우연적 응집도 (Coincidental Cohesion)**|모듈 내부 요소들이 아무 관련 없이 단순히 묶여 있는 경우|임의로 여러 기능을 한 함수에 넣음|
|**논리적 응집도 (Logical Cohesion)**|유사한 성격의 기능을 한 모듈에 모아둔 경우|입력 처리 함수에 키보드, 마우스, 파일 입력을 모두 처리|
|**시간적 응집도 (Temporal Cohesion)**|특정 시점에 실행되는 기능들을 한 모듈에 모아둔 경우|프로그램 시작 시 초기화 작업 모듈|
|**절차적 응집도 (Procedural Cohesion)**|특정 절차에 따라 순차적으로 수행되는 기능들을 모아둔 경우|파일 열기 → 읽기 → 닫기 절차를 한 모듈에 묶음|
|**통신적 응집도 (Communicational Cohesion)**|동일한 입력이나 출력을 사용하는 기능들을 모아둔 경우|같은 파일을 읽고 처리하는 여러 기능을 한 모듈에 묶음|
|**순차적 응집도 (Sequential Cohesion)**|한 기능의 출력이 다음 기능의 입력으로 사용되는 경우|데이터 처리 후 그 결과를 바로 출력하는 모듈|
|**기능적 응집도 (Functional Cohesion)**|모듈이 하나의 기능만을 수행하도록 구성된 경우|"합계 계산"만 수행하는 함수|
