> **Clang** (클랭)은 LLVM 프로젝트의 하위 프로젝트로, **C, C++, Objective-C, Objective-C++ 언어를 위한 공식 프론트엔드**(Frontend)입니다.


Clang은 기존의 지배적인 컴파일러였던 **GCC(GNU Compiler Collection)의 한계를 극복**하기 위해 설계되었습니다.

- **GCC의 한계:**
    
    - **일체형(Monolithic) 구조:** 컴파일러의 각 부분이 너무 긴밀하게 연결되어 있어, 파서(parser)나 의미 분석기 같은 특정 기능만 따로 떼어내어 다른 도구를 만들기 매우 어려웠습니다.
        
    - **라이선스:** GPL 라이선스를 채택하여, 일부 상용 소프트웨어에 통합하기에 제약이 있었습니다.
        
    - **성능 및 오류 메시지:** 당시 GCC는 컴파일 속도가 상대적으로 느리고, 오류 메시지가 사용자에게 불친절하다는 평이 있었습니다.
        
- **Clang의 설계 목표:**
    
    - **모듈화된 라이브러리 기반 설계:** 처음부터 재사용성을 염두에 두고 설계되었습니다. 파서, AST 생성기 등이 모두 독립적인 라이브러리로 제공되어, 누구나 이 부품들을 가져다 새로운 개발 도구를 쉽게 만들 수 있도록 했습니다.
        
    - **빠른 컴파일 속도와 적은 메모리 사용량:** 더 효율적인 알고리즘을 사용하여 성능을 개선하는 것을 목표로 했습니다.
        
    - **친절하고 정확한 오류 메시지:** 개발자가 오류를 쉽게 이해하고 수정할 수 있도록, 최대한 상세하고 유용한 진단 메시지를 제공하는 것을 최우선 과제로 삼았습니다.

## Clang의 핵심 특징 및 장점

#### **1. 모듈화된 라이브러리 기반 아키텍처**

Clang의 가장 중요하고 혁신적인 특징입니다. Clang은 하나의 거대한 프로그램이 아니라, 여러 **C++ 라이브러리의 집합체**입니다. 예를 들어, `libclang`과 `libtooling` 라이브러리를 사용하면 단 몇 줄의 코드로 C++ 소스 코드를 파싱하고 AST를 자유롭게 다룰 수 있습니다. 이것이 바로 **Clang-Tidy, Clang Static Analyzer, clangd(언어 서버)** 같은 수많은 강력한 도구들이 탄생할 수 있었던 배경입니다.


#### **2. 표현력이 풍부한 진단 메시지 (Expressive Diagnostics) 🩺**

Clang은 오류 메시지가 매우 친절하기로 유명합니다.

- **정확한 위치 표시:** 코드에서 오류가 발생한 정확한 위치를 `^`와 `~` 기호로 명확하게 알려줍니다.
    
- **오류 수정 제안:** 단순한 오류 보고를 넘어, 가능한 수정 방안을 제시합니다. 예를 들어, `if (a = b)`와 같은 실수를 `if (a == b)`로 수정하라고 제안해 줍니다.
    
- **컬러 하이라이팅:** 터미널에서 오류 메시지를 색상으로 구분하여 가독성을 높여줍니다.

- warning(보라색) : Clang-Tidy등 커스텀 체커 관련 메시지
- error(빨간색) : Clang 컴파일러의 diagnostic 엔진에서 경고하는 오류
- note(파란색) : 기타 알려야 할 정보를 위해 사용할수 있는 메시지

#### **3. 빠른 컴파일 속도 및 효율성**

Clang은 소스 코드를 파싱하고 AST를 생성하는 과정에서 GCC보다 더 빠르고 메모리를 적게 사용하도록 설계되었습니다. 이는 대규모 프로젝트를 빌드할 때 큰 이점을 제공합니다.

#### **4. 강력한 도구 생태계**

라이브러리 기반 설계 덕분에 Clang을 기반으로 한 다양한 도구들이 존재합니다.

- **Clang-Tidy:** 코드 스타일 및 버그 패턴을 검사하고 자동 수정합니다.
    
- **Clang Static Analyzer:** 깊이 있는 경로 분석을 통해 런타임 버그를 찾습니다.
    
- **clang-format:** 코딩 스타일을 일관되게 맞춰주는 포맷터입니다.
    
- **clangd:** Visual Studio Code 같은 최신 편집기에서 코드 자동 완성, 정의 이동 등의 기능을 제공하는 언어 서버입니다.

이때 우리가 원하는것은 CSA(Clang_)
